## 二、本课整体理解

Lesson 6 是 JavaScript 真正开始“会判断”的一课。

前面几课你已经学会：
- 数字
- 字符串
- 变量
- 按钮点击后执行代码

但这些代码仍然缺少一个关键能力：

> 根据不同情况，执行不同结果。

例如：
- 如果年龄够大，就允许开车
- 如果购物车里有商品，就显示提示
- 如果电脑出了石头，而你出了布，就算你赢

这就是布尔值和 `if` 语句的意义。

这一课最重要的变化是：

> 程序开始从“顺着写下去”变成“根据条件分支执行”。

这会让你的代码第一次有“决策能力”。

## 三、什么是布尔值 Boolean

### 1. 布尔值的定义

对应视频时间：`00:01:44`

布尔值是 JavaScript 中的一种值类型，用来表示：

> 某件事是真的还是假的。

---

### 2. 只有两个布尔值

对应视频时间：`00:01:58`

JavaScript 中布尔值只有两个：

```js
true
false
```

注意：
- 它们不是字符串
- 不要写成 `'true'` 或 `'false'`

---

### 3. 布尔值和字符串的区别

对应视频时间：`00:03:44`

```js
true
'true'
```

这两个完全不是一回事：
- 第一个是布尔值
- 第二个是字符串

这点非常重要，因为判断条件时通常需要的是布尔值，不是文本。

---

## 四、比较运算符

### 1. 比较会产生布尔值

对应视频时间：`00:02:49`

例如：

```js
3 < 5
3 > 5
```

这些表达式的结果不是数字，而是布尔值：
- `true`
- `false`

---

### 2. 常见比较运算符

对应视频时间：`00:04:37`

课程讲到的主要有：

- `>`：大于
- `<`：小于
- `>=`：大于等于
- `<=`：小于等于
- `===`：严格相等
- `!==`：严格不相等

这些是后面条件判断最核心的工具。

---

### 3. 为什么推荐用 `===` 而不是 `==`

对应视频时间：`00:05:14`

课程特别强调：

> JavaScript 里检查相等，优先使用 `===`。

因为 `==` 会发生自动类型转换，例如：

```js
5 == '5'
```

可能得到 `true`，这会带来歧义。

而：

```js
5 === '5'
```

会更严格地比较类型和值，更安全。

---

### 4. 为什么推荐用 `!==` 而不是 `!=`

对应视频时间：`00:06:22`

原因和上面一样：  
避免类型自动转换带来的混乱。

所以本课程给出的最佳实践是：
- 相等：用 `===`
- 不相等：用 `!==`

---

### 5. 比较运算符的优先级

对应视频时间：`00:06:58`

课程讲到：

> 比较运算符的优先级低于数学运算。

例如：

```js
3 > 5 - 5
```

JavaScript 会先算：

```js
5 - 5
```

再做比较。

这点在混合表达式里很重要。

---

## 五、`if` 语句

### 1. `if` 的作用

对应视频时间：`00:07:36`

`if` 语句让程序根据条件决定是否执行某段代码。

基本结构：

```js
if (condition) {
  // 条件为 true 时执行
}
```

---

### 2. 条件位置放什么

对应视频时间：`00:11:02`

圆括号里不一定非得直接写 `true` 或 `false`。  
也可以放任何“最终会得到布尔值”的代码，例如比较表达式：

```js
if (age >= 16) {
  console.log('You can drive');
}
```

---

### 3. `else`

对应视频时间：`00:09:09`

如果 `if` 条件不成立，就执行 `else` 里的代码：

```js
if (age >= 16) {
  console.log('You can drive');
} else {
  console.log('You cannot drive');
}
```

这就形成了两个分支。

---

### 4. `else if`

对应视频时间：`00:13:40`

当你不只想判断两种情况，而是多种情况时，可以用：

```js
if (...) {
} else if (...) {
} else {
}
```

课程用“年龄快够开车”的例子演示了第三种情况。

这就是多分支判断的基础。

---

### 5. 分支 `branch`

对应视频时间：`00:12:48`

课程把每个大括号里的代码块叫做分支 `branch`。  
你可以把它想成程序走到这里时可能分叉成不同路径。

这个思路非常重要，因为以后：
- if
- loop
- function

都会引出“代码块”和“作用域”的概念。

---

## 六、`if` 语句的语法细节

### 1. 单行时大括号有时可省，但不推荐滥省

对应视频时间：`00:12:59`

课程提到如果分支里只有一行代码，大括号有时可以省略。  
但如果有多行代码，大括号就是必须的。

入门阶段最稳妥的建议是：

> 尽量保留大括号。

这样更清晰，也更不容易出错。

---

### 2. `else` 是可选的

对应视频时间：`00:12:18`

不是每个 `if` 都必须带 `else`。  
如果你只关心“条件成立时做一件事”，那只写 `if` 也完全可以。

---

## 七、在条件中使用变量

### 1. 变量能让条件更可维护

对应视频时间：`00:15:21`

课程把年龄单独保存进变量：

```js
const age = 15;
```

然后在多个比较里重复使用它。

这样做的好处是：
- 改值时只改一处
- 代码更清晰
- 更接近真实项目写法

---

### 2. 变量 + 条件判断是后面大量代码的基础

从这一课开始，你会越来越多看到这种组合：

```js
const something = ...;

if (something ...) {
  ...
}
```

这几乎是所有业务逻辑的起点。

---

## 八、逻辑运算符

### 1. `&&`：并且

对应视频时间：`00:25:45`

`&&` 用来组合两个条件，只有两边都为真，结果才为真。

例如：

```js
randomNumber >= 0 && randomNumber < 1 / 3
```

这表示：

> 这个数字既要大于等于 0，又要小于 1/3。

---

### 2. `||`：或者

对应视频时间：`00:31:03`

`||` 的含义是：

> 两边至少有一边为真，整体就为真。

例如：

```js
true || false
```

结果是 `true`。

---

### 3. `!`：取反

对应视频时间：`00:31:57`

`!` 会把布尔值翻转：

```js
!true  // false
!false // true
```

这在判断“不是某种情况”时非常常见。

---

### 4. 逻辑运算符的优先级

对应视频时间：`00:27:50`

课程提到，逻辑运算符优先级低于：
- 数学运算
- 比较运算

所以通常会先算数学和比较，再用 `&&` / `||` 组合结果。

## 九、石头剪刀布项目

### 1. 先写算法，再写代码

对应视频时间：`00:20:36`

课程明确提出了“算法”这个概念：

> 算法就是完成任务的一组步骤。

石头剪刀布项目的步骤是：

1. 玩家点击一个按钮
2. 电脑随机出一个动作
3. 比较玩家动作和电脑动作
4. 得出输赢结果
5. 显示结果

这个顺序非常重要，因为它训练的是“先拆步骤，再写代码”的能力。

---

### 2. `Math.random()`

对应视频时间：`00:22:01`

课程引入了：

```js
Math.random()
```

它会生成一个：

> 大于等于 0 且小于 1 的随机数。

每次运行，结果通常都不同。

---

### 3. 用随机数生成电脑动作

对应视频时间：`00:23:20`

课程把 `0` 到 `1` 的区间分成三段：

- `0` 到 `1/3`：石头
- `1/3` 到 `2/3`：布
- `2/3` 到 `1`：剪刀

这样就能把随机数映射成电脑动作。

这是本课非常典型的“数学 + 条件判断”结合案例。

---

### 4. 为什么用 `<` 而不是 `<=`

对应视频时间：`00:30:13`

课程特地说明了：

> 为了避免区间重叠，通常一边写 `>=`，另一边写 `<`。

这样边界更清晰，不会一个数同时落进两个区间。

---

### 5. 比较玩家动作和电脑动作

对应视频时间：`00:38:00`

课程先以“玩家选择 Rock”为例，用 `if / else if / else` 比较：
- 电脑是 Rock：平局
- 电脑是 Paper：你输
- 电脑是 Scissors：你赢

后面再分别复制调整到 Paper 和 Scissors 按钮。

这是典型的分支判断。

### 6. 用变量保存结果和电脑动作

对应视频时间：`00:39:10` 到 `00:41:20`

课程把：
- `computerMove`
- `result`

都放进变量里。

这是非常正确的做法，因为：
- 后面需要重复使用
- 可以统一插入到模板字符串
- 结构比直接写死在 `alert()` 里清晰得多

---

### 7. 用模板字符串显示结果

对应视频时间：`00:41:35`

最后的弹窗不是写死的，而是类似：

```js
`You picked rock. Computer picked ${computerMove}. ${result}`
```

这一步把前几课知识串起来了：
- 变量
- 模板字符串
- 条件判断
- 点击事件

## 十、作用域 Scope

### 1. 什么是作用域

对应视频时间：`00:34:20`

课程解释说：

> 作用域限制变量存在和可访问的位置。

最直观的理解是：

> 在某个大括号里创建的变量，通常只在这个大括号里有效。

---

### 2. 为什么在 `if` 内创建的变量外面访问不到

对应视频时间：`00:34:20` 到 `00:36:20`

如果你在某个 `if` 分支内部：

```js
const computerMove = 'rock';
```

那它通常只能在这个分支内部使用。  
外面访问不到，就会报“未定义”。

所以课程后面把变量挪到了 `if` 语句外部，先创建，再在分支里重新赋值。

---

### 3. 为什么作用域是好事

课程也给了动机：

> 作用域可以帮助避免命名冲突。

否则在大型代码里，变量名很容易互相打架。

这也是现代 JavaScript 设计里非常核心的机制。

---

### 4. 为什么 `var` 不推荐

对应视频时间：`00:37:12`

课程借这个场景再次提醒：

> `var` 不遵守这里这种更合理的块级作用域规则。

这也是前一课里为什么建议优先用：
- `const`
- `let`

而不是 `var`。

## 十一、Truthy 和 Falsy

### 1. `if` 不只接受布尔值

对应视频时间：`00:46:49`

课程强调，`if` 条件里不一定非要放 `true` 或 `false`。  
它也能处理其他值，例如数字和字符串。

这些值会被 JavaScript 当成：
- truthy
- falsy

---

### 2. 什么是 truthy

truthy 的意思是：

> 虽然它不是布尔值 `true`，但在条件判断里表现得像 `true`。

例如：

```js
5
```

在 `if (5)` 里会当成真。

---

### 3. 什么是 falsy

对应视频时间：`00:48:03`

falsy 的意思是：

> 虽然它不是布尔值 `false`，但在条件判断里表现得像 `false`。

课程给出的 falsy 值列表是：

- `false`
- `0`
- `''`
- `NaN`
- `undefined`
- `null`

除此之外的大多数值都可以看作 truthy。

---

### 4. 为什么 truthy/falsy 有用

对应视频时间：`00:48:55`

课程举了购物车数量的例子：

如果 `cartQuantity` 是 `0`，那它就是 falsy。  
如果是其他非零数字，它通常就是 truthy。

所以有时你可以把：

```js
if (cartQuantity > 0)
```

简写成：

```js
if (cartQuantity)
```

这是 JavaScript 很常见的写法。

---

### 5. `NaN`

对应视频时间：`00:50:16`

`NaN` 表示：

> Not a Number

也就是某个结果不是合法数字。

课程用“字符串除以数字”这种无效运算举了例子。

---

### 6. `undefined`

对应视频时间：`00:51:18`

`undefined` 表示：

> 没有值。

例如：

```js
let variable1;
```

如果你只创建变量但不给它赋值，它通常就是 `undefined`。

---

## 十二、`if` 的几种快捷写法

### 1. 三元运算符 `? :`

对应视频时间：`00:52:48`

三元运算符是 `if / else` 的简写形式：

```js
condition ? valueIfTrue : valueIfFalse
```

课程还演示了把它直接保存到变量中。

这在“根据条件得到一个值”时非常方便。

---

### 2. 守卫运算符 Guard Operator

对应视频时间：`00:55:02`

课程把 `&&` 的短路行为拿来当作一种简写：

```js
condition && doSomething
```

含义类似于：

> 如果左边为真，才继续执行右边。

这是一种简洁但需要你已经熟悉逻辑运算符的写法。

---

### 3. 默认运算符 Default Operator

对应视频时间：`00:57:33`

课程把 `||` 用来设置默认值：

```js
const currency = userChoice || 'USD';
```

意思是：
- 如果左边有有效值，就用左边
- 如果左边是 falsy，就退回右边默认值

这在处理“用户没填值”“参数可选”时非常常见。

---

## 十三、本课最容易出错的地方

### 1. 把 `true` / `false` 写成字符串

```js
'true'
'false'
```

这不是布尔值。

---

### 2. 相等比较还在用 `==`

这节课课程已经明确建议优先用 `===`。

---

### 3. 条件写错区间边界

在石头剪刀布项目里，随机数分段时如果边界写重叠，就会出 bug。

---

### 4. 在 `if` 分支里创建变量，外面还想继续用

这就是典型的作用域错误。

---

### 5. truthy/falsy 理解混乱

例如：
- `0` 是 falsy
- `'0'` 是 truthy

很多新手第一次会混淆。

---

### 6. 三元运算符滥用

三元适合简短情况，如果逻辑很复杂，普通 `if` 往往更清晰。

---

## 十四、本课最重要的知识点总结

1. 布尔值只有 `true` 和 `false`
2. 比较表达式通常返回布尔值
3. `if` 让程序根据条件执行不同分支
4. `else if` 可以处理多种情况
5. JavaScript 中相等判断优先用 `===`
6. 逻辑运算符有 `&&`、`||`、`!`
7. `Math.random()` 能生成 0 到 1 之间的随机数
8. 作用域决定变量在哪些位置可用
9. `if` 能接受 truthy / falsy，不只接受布尔值
10. `0`、`''`、`NaN`、`undefined`、`null` 都是 falsy
11. 三元、守卫、默认运算符都是条件判断的简写
12. 写代码前先拆算法，是非常重要的习惯

## 十七、给初学者的最终提醒

这节课最重要的收获，不是记住几个新符号，而是开始理解：

> 程序可以根据情况做不同决定。

从这一课开始，JavaScript 不再只是“顺着往下执行”的脚本，而是变成了能够：
- 判断
- 分支
- 组合条件
- 根据结果采取不同行动

这正是“编程逻辑”真正开始形成的地方。

如果你把布尔值、比较运算符、`if`、逻辑运算符、作用域、truthy/falsy 这些点真正掌握住，后面学函数、DOM 交互和项目逻辑时会轻松很多。
