## 五、CSS 选择器、属性和值

### 1. 选择器是什么

对应视频时间：`00:03:00`

例如：

```css
button {
  border: none;
}
```

这里 `button` 就是 CSS 选择器。  
它告诉浏览器：要给哪些元素加样式。

---

### 2. 属性和值

对应视频时间：`00:04:35`

例如：

```css
border: none;
```

这里：

1. `border` 是属性
2. `none` 是值

它们之间用冒号分隔，最后以分号结束。

这和对象里的“属性-值对”很像，但它是 CSS 语法。

---

### 3. 类选择器

对应视频时间：`00:03:35`

如果只想给某个特定元素加样式，就可以先给元素加 `class`：

```html
<button class="subscribe-button">Subscribe</button>
```

然后在 CSS 中：

```css
.subscribe-button {
  border: none;
}
```

前面的点 `.` 表示按 class 选择。

---

## 六、一个元素可以有多个 class

### 1. 为什么要多个 class

对应视频时间：`00:03:18`

课程里特别强调：

一个元素可以同时有多个 class，用空格分开。

例如：

```html
<button class="js-subscribe-button subscribe-button">
  Subscribe
</button>
```

这里：

1. `js-subscribe-button` 给 JavaScript 用
2. `subscribe-button` 给 CSS 用

---

### 2. 为什么要分开

这是一个很好的工程习惯：

1. JS 类名用于功能选择
2. CSS 类名用于界面样式

这样功能和样式职责更清晰，不容易混乱。

## 八、用 JavaScript 改变 CSS 状态

### 1. 问题背景

对应视频时间：`00:11:30`

前一课订阅按钮点击后只改了文字。  
这一课想让它点击后连背景色和文字色一起变。

这就需要：

JavaScript 去改变元素的 class。

---

### 2. `classList`

对应视频时间：`00:12:10`

每个 HTML 元素对象都有一个属性：

```js
element.classList
```

它让我们操作元素的 class 列表。

---

### 3. `classList.add()`

对应视频时间：`00:12:50`

```js
buttonElement.classList.add('is-subscribed');
```

表示给这个元素新增一个 class。

然后 CSS 再专门给这个类写样式：

```css
.is-subscribed {
  background-color: rgb(240, 240, 240);
  color: black;
}
```

---

### 4. `classList.remove()`

对应视频时间：`00:18:28`

```js
buttonElement.classList.remove('is-subscribed');
```

表示把这个类移除。

所以点击按钮时：

1. 加上类 -> 应用新样式
2. 移除类 -> 回到原样式

这就是最基础的“状态切换式样式控制”。

---

### 5. 注意这里不要写点 `.`

课程里专门提醒：

```js
classList.add('is-subscribed');
```

传入的是类名本身，不要写成：

```js
classList.add('.is-subscribed');
```

点只在 CSS 选择器和 `querySelector()` 中有意义。

## 十六、本课应该顺手补充理解的内容

### 1. CSS 和 JavaScript 不冲突，而是互补

课程里最重要的一个思路是：

1. CSS 负责定义样式规则
2. JavaScript 负责控制什么时候应用这些规则

这就是为什么 `classList.add()` / `remove()` 这么重要。

---

### 2. 动态 class 切换是前端的核心模式之一

比如：

1. 订阅 / 已订阅
2. 展开 / 收起
3. 激活 / 未激活
4. 错误 / 正常

很多界面状态切换，本质上都是改 class。

---

### 3. 图片、字体、布局、按钮风格，本质上都是“界面资源和规则”

这一课虽然看起来像“调样式”，但其实你在学：

1. 资源怎么接入页面
2. 视觉状态怎么统一
3. 页面结构怎么和样式配合

---

## 十七、常见错误与易混点

### 1. 在 `classList.add()` 里写 `.class-name`

错误：

```js
buttonElement.classList.add('.is-subscribed');
```

正确：

```js
buttonElement.classList.add('is-subscribed');
```

---

### 2. 忘记一个元素可以有多个 class

很多初学者会误以为一个元素只能有一个 class。  
其实完全可以同时有多个。

---

### 3. 把 `padding` 和 `margin` 混淆

记忆方法：

1. `padding` 在里面
2. `margin` 在外面

---

### 4. 图片移动到文件夹后没改路径

如果图片原来在当前目录，后来移动到 `images/`，那 `src` 一定要一起改。

---

### 5. 只改 JavaScript 文本，不改对应 class

在订阅按钮里，如果只改文字不改 class，视觉状态就不会同步变化。

---

### 6. 以为复制最终设计颜色只能手写猜

课程里已经展示了更实际的做法：  
用开发者工具直接查看和复制样式。

---

## 十八、本课最重要的结论

这一课最应该牢固掌握的是：

1. HTML、CSS、JavaScript 是协作关系
2. CSS 用选择器决定给谁加样式
3. class 是连接 HTML、CSS、JavaScript 的关键机制
4. 一个元素可以同时有多个 class
5. `padding` 是内边距，`margin` 是外边距
6. `classList.add()` / `remove()` 可以动态切换样式状态
7. 图片通过 `<img>` 和 `src` 加载
8. 文件位置变了，路径也要一起改
9. 大一点的项目应该把 HTML / CSS / JS 分文件组织

### 2. class 切换练习

1. 创建一个按钮
2. 点击时添加一个 class
3. 再点击时移除这个 class
4. 用 CSS 让这两个状态视觉明显不同

### 3. 图片练习

1. 在页面中插入一张图片
2. 给图片加 class
3. 用 CSS 控制图片高度
4. 把图片移进文件夹，再改路径让它重新显示

### 4. 文件组织练习

1. 把一个小网页中的 CSS 挪到单独 `.css` 文件
2. 把 JavaScript 挪到单独 `.js` 文件
3. 确保页面还能正常运行
