## 二、本课整体理解

Lesson 17 是 JavaScript 课程里第一次系统讲“编程风格”的一课。

前面你主要写的是：
- 数据
- 函数
- 模块
- 页面渲染逻辑

这一课开始讨论：

> 我们应该怎样组织这些代码，才能让它更像“对象”而不是一堆分散函数。

这一课最重要的核心思想可以概括成两句：

> OOP 的重点不是类语法本身，而是“把相关数据和行为放在一起”。  
> 类、继承、多态这些能力，本质上都是为了组织和复用代码。

## 六、`this` 在对象方法里是什么意思

### 1. `this` 指向外部对象

对应视频时间：`00:08:02`

课程这里给出了本课最核心的 `this` 规则之一：

> 在对象方法里，`this` 指向这个外部对象。

所以你在 `cart` 对象的方法里写：

```js
this.cartItems
```

意思就是访问：

> 当前这个购物车对象自己的 `cartItems`

---

### 2. 为什么 `this` 比直接写对象名更好

对应视频时间：`00:08:50`

如果你在方法里直接写对象变量名，比如：

```js
cart.cartItems
```

那么对象名一旦改掉，方法内部代码也要跟着改。

而 `this` 更稳定，因为它表达的是：

> 当前对象自己

但如果你要创建多个类似对象，比如：
- 普通购物车

### 2. 工厂函数的思路

对应视频时间：`00:25:29`

### 3. 为什么参数 `localStorageKey` 很重要

## 十一、构造函数 Constructor

### 1. 构造函数的作用

对应视频时间：`00:41:25`

课程把构造函数介绍为：

> 对象创建后自动运行的初始化代码位置。

通常用于：
- 设置初始属性
- 调用初始化方法
- 做准备工作

---

### 2. 为什么构造函数能让代码更干净

对应视频时间：`00:41:48`

如果你在类外面每次都写：
- 设置本地存储 key
- 再调用加载逻辑

那初始化步骤分散在外部。

### 3. 构造函数参数

```js

```

然后实例化时传入：

```js
new Cart('cart-business')
```

这样每个实例都能带着自己的初始化数据。

---

### 4. 构造函数的规则

对应视频时间：`00:45:50`

课程提到两个很重要的规则：

## 十三、私有属性和私有方法

### 1. 为什么要有私有成员

对应视频时间：`00:47:31`

课程指出一个现实问题：

如果类外部可以随便改内部关键字段，比如：

```js
cart.localStorageKey = 'something-else';
```

就可能把对象搞坏。

所以类希望保护某些内部实现细节。

---

### 2. 私有属性写法

对应视频时间：`00:47:31`

JavaScript 类里可以用：

```js
#localStorageKey
```

前面的 `#` 表示：

> 私有字段

只能在类内部访问。

---

### 3. 公共属性是什么

对应视频时间：`00:49:00`

没有 `#` 的普通属性就是公共属性：

> 可以在类外部访问。

例如：

```js
cart.cartItems
```

---

### 4. 私有方法

对应视频时间：`00:50:44`

方法也能私有化：

```js
#loadFromStorage() {
  ...
}
```

这表示这个方法只是类内部实现细节，不希望类外部调用。

---

### 5. 私有成员解决了什么问题

它主要解决：
- 避免外部误改
- 限制 API 暴露范围
- 让类的使用边界更清晰

这是封装思想的一部分。

`map()` 最适合这种场景：

> 把一个数组中的每个值“转换成另一种形式”，同时保留数组结构。

这里就是：
- 普通对象数组
- 转成类实例数组

## 十六、继承 Inheritance

### 1. 为什么要讲继承

对应视频时间：`01:15:20`

课程开始讨论：

现实中不是只有“产品”这一种抽象对象，还有：
- 服装

这些都是产品，但又更具体。

如果每个更具体的类都复制一份 `Product` 的代码，会很重复。

### 2. 继承是什么

对应视频时间：`01:15:20`

继承就是：

> 一个类获取另一个类的属性和方法，以复用代码。

例如：

```js
class Clothing extends Product {
}
```

表示：

> `Clothing` 继承 `Product`

---

### 3. 父类和子类

对应视频时间：`01:16:40`

课程里：
- `Product` 是父类 `parent class`
- `Clothing` 是子类 `child class`

子类会继承父类已有能力，再加上自己额外能力。

## 十九、方法重写 Method Overriding

### 1. 什么是方法重写

对应视频时间：`01:34:56`

如果父类有一个方法：

```js
extraInfoHTML()
```

子类也定义了同名方法，那么：

> 子类版本会覆盖父类版本。

这就叫方法重写。

---

### 2. 课程里的实际例子

父类 `Product`：
- `extraInfoHTML()` 返回空字符串

子类 `Clothing`：
- `extraInfoHTML()` 返回尺码表链接 HTML

这样同样调用：

```js
product.extraInfoHTML()
```

不同类实例会给出不同结果。

---

## 二十、多态 Polymorphism

### 1. 什么是多态

对应视频时间：`01:36:08`

课程给出的核心理解是：

> 我们可以调用同一个方法，而不用事先知道对象具体属于哪个类。

比如这里：

```js
product.extraInfoHTML()
```

你不一定先判断它是 `Product` 还是 `Clothing`。

类自己决定这次应该返回什么。

---

### 2. 多态为什么能减少 `if/else`

对应视频时间：`01:36:40`

如果不用多态，你可能会写：
- 如果是服装，就渲染尺码表
- 如果是普通产品，就不渲染

有了多态后：
- 统一调用 `extraInfoHTML()`
- 每个类自己负责实现

这会让页面代码更干净，也更容易扩展。

## 二十二、`this` 的详细规则

### 1. 顶层模块中的 `this`

对应视频时间：`01:43:48`

课程提到，在 JavaScript 模块中，顶层直接写：

```js
console.log(this);
```

通常是：

```js
undefined
```

这和早期非模块脚本里的行为不同。

---

### 2. 在对象属性初始化时，`this` 不会像你想的那样工作

对应视频时间：`01:44:40`

课程举例说明：

在对象字面量的属性初始化阶段，`this` 并不会自动指向这个对象，因为对象本身还没完成创建。

所以这种写法往往不对：

```js
const obj = {
  a: 2,
  b: this.a
};
```

---

### 3. 普通函数里的 `this`

对应视频时间：`01:46:08`

在普通函数里，如果没有对象调用上下文：

```js
function logThis() {
  console.log(this);
}
```

在模块环境中通常是：

```js
undefined
```

---

### 4. `call()` 改变 `this`

对应视频时间：`01:47:20`

课程用：

```js
logThis.call('hello');
```

说明：

> 可以手动指定函数执行时的 `this`。

`call()` 的第一个参数就是这次运行时的 `this` 值。

---

### 5. 箭头函数不会绑定自己的 `this`

对应视频时间：`01:48:52`

这是本课关于 `this` 的最重要规则之一：

> 箭头函数不会创建自己的 `this`，它会直接使用外层的 `this`。

这叫词法继承 `lexical this`。

---

### 6. 为什么 `forEach` 里常用箭头函数

对应视频时间：`01:50:56`

课程举的典型问题是：

如果你在对象方法里再写一个普通函数，比如 `forEach(function () {})`，那么里面的 `this` 可能不再指向原对象。

但如果你改成箭头函数：

```js
forEach(() => {
  ...
})
```

它会继承外层方法的 `this`，从而保持正确。

## 二十三、本课应该补充理解的相关知识点

### 1. 封装 Encapsulation

课程虽然没有反复突出这个术语，但你其实已经接触到了：

> 把数据和操作数据的方法放在一起，并隐藏一部分内部细节。

比如：
- `cartItems` 和 `addToCart()` 放在一起
- `#localStorageKey` 设为私有

这就是封装思想。

---

### 2. 工厂函数 vs 类

课程先用了函数生成对象，再升级到类。

你应该补充理解：

- 工厂函数：更灵活，语法更普通
- 类：更适合表达“这是一个对象模板”

两者都能生成对象，只是风格不同。

---

### 3. OOP 不是“必须比过程式更好”

课程最后也明确说了：

> 选哪种风格，取决于你自己和团队。

这很重要，因为很多初学者会误以为：

> 学了 class 就比函数“更高级”

其实不是。只是不同场景有不同取舍。

## 二十四、常见错误整理

### 1. 在对象方法里直接写对象变量名而不是 `this`

这样对象名一改，内部代码就容易坏。

---

### 2. 忘记 `new`

创建类实例时如果忘写：

```js
new Cart()
```

而只写 `Cart()`，通常会出问题。

---

### 3. 在子类构造函数里忘记 `super()`

子类需要继承父类初始化逻辑时，忘记调用 `super()` 会导致构造流程不完整。

---

### 4. 把私有字段当公共字段来访问

例如在类外部写：

```js
cart.#localStorageKey
```

这是不允许的。

---

### 5. 误以为箭头函数里的 `this` 会指向当前对象

箭头函数不会自己绑定 `this`，它继承外层作用域的 `this`。

---

### 6. 用继承时复制粘贴父类代码

继承的目的就是减少重复。如果你已经 `extends` 了父类，还把父类逻辑整段再复制一遍，通常说明没真正用好继承。

---

## 二十五、本课最重要的知识点总结

你至少要真正掌握下面这些：

1. OOP 是一种把代码组织成对象的编程风格
2. 过程式编程和 OOP 是两种不同组织方式
3. 对象里的函数叫方法
4. `this` 在方法里通常指向调用该方法的对象
5. 函数也能生成对象，但类更适合专门做这件事
6. 类是对象生成器，`new` 会创建实例
7. `constructor` 用来做初始化
8. 私有字段和私有方法用来保护内部实现
9. 继承让子类复用父类代码
10. `super()` 能调用父类构造函数或父类方法
11. 方法重写表示子类替换父类同名方法
12. 多态表示同名方法可以在不同类中表现不同
13. 箭头函数不会改变 `this`
