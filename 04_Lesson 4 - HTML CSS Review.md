## 六、HTML 元素与标签

### 1. 第一个按钮元素

对应视频时间：`00:03:19`

课程的第一个 HTML 例子是按钮：

```html
<button>Hello</button>
```

这段代码告诉浏览器：
- 创建一个按钮
- 按钮里面显示文字 `Hello`

---

### 2. 段落元素 `<p>`

对应视频时间：`00:05:16`

课程接着演示了段落：

```html
<p>Text paragraph</p>
```

它会创建一段普通文本。

这再次说明 HTML 的重点是：

> 告诉浏览器页面里要有哪些内容。

---

### 3. 什么是 HTML 元素

对应视频时间：`00:06:30`

课程把按钮、段落这些称为 HTML 元素。

例如：
- 按钮元素
- 段落元素

都是网页的组成部分。

---

### 4. 什么是开始标签和结束标签

对应视频时间：`00:06:44`

以按钮为例：

```html
<button>
</button>
```

其中：
- `<button>` 是开始标签
- `</button>` 是结束标签

开始标签表示元素开始，结束标签表示元素结束。

中间夹的内容，就是这个元素的内容。

---

### 5. 元素里不只能放文本，也能放别的元素

对应视频时间：`00:07:34`

课程演示了在段落里放按钮，这叫：

> 嵌套 `nesting`

例如：

```html
<p>
  Text paragraph
  <button>Hello</button>
</p>
```

这说明 HTML 不是平铺罗列，而是一个可以嵌套的结构。

## 七、HTML 中空格和换行的规则

### 1. 多个空格会被合并成一个

对应视频时间：`00:08:00`

课程强调：

> 在 HTML 普通文本里，连续多个空格通常只显示为一个空格。

这意味着你在代码里拼很多空格，网页上不一定真的显示那么多。

---

### 2. 换行通常也被当成空格

对应视频时间：`00:08:40`

HTML 代码里的换行，不会自动在页面里生成新的一行视觉效果。  
浏览器一般会把它也当成空白的一部分处理。

这就是为什么你可以放心用换行来整理代码，而不一定会破坏页面显示。

---

### 3. 为什么这条规则反而很有用

对应视频时间：`00:09:20`

正因为空格和换行在 HTML 里不敏感，开发者才能：
- 把代码拆成多行
- 添加缩进
- 让结构更清晰

这是 HTML 可读性的重要基础。

## 八、CSS 基础

### 1. CSS 是什么

对应视频时间：`00:10:43`

课程说得很清楚：

> CSS 改变网站的外观。

也就是说，HTML 负责“有什么”，CSS 负责“长什么样”。

---

### 2. `<style>` 元素

对应视频时间：`00:11:10`

在 HTML 文件里，课程先使用：

```html
<style>
</style>
```

来写 CSS。

这个元素本身通常不会直接显示在页面上，它的作用是承载样式代码。

### 3. CSS 选择器

对应视频时间：`00:11:46`

例如：

```css
button {
  background-color: red;
}
```

这里的：

```css
button
```

叫选择器 `selector`，它表示：

### 4. CSS 属性和值

对应视频时间：`00:13:20`

例如：

```css

```

这里：
- `background-color` 是属性 `property`
- `red` 是值 `value`

中间用冒号 `:` 分隔，结尾通常用分号 `;` 结束。

---

### 5. 课程中的几个典型样式

对应视频时间：`00:12:14` 到 `00:13:13`

### 1. 什么是 HTML 属性

对应视频时间：`00:14:44`

> 属性会改变元素的行为。

属性通常写在开始标签里，例如：

### 2. `title` 属性和工具提示

课程使用了：


把鼠标移上去时，浏览器会显示一个小提示框，这就是 tooltip。

---

### 3. 属性的语法规则

对应视频时间：`00:16:39`

属性的基本结构是：

```html
attributeName="value"
```

几个关键点：
- 属性写在开始标签里
- 属性名和值之间用 `=`

## 十、`class` 属性与类选择器

### 1. `class` 是给元素打标签

对应视频时间：`00:17:02`

课程把 `class` 解释得很实用：

> `class` 有点像给元素贴标签。

例如：

```html
<button class="red-button">Hello</button>
```

这表示这个按钮被打上了 `red-button` 这个类名。

---

### 2. 用 `.class-name` 选择有该类的元素

对应视频时间：`00:18:08`

在 CSS 中：

```css
.red-button {
  background-color: red;
}
```

开头的点 `.` 表示：

> 这是一个类选择器。

它会选中所有 `class="red-button"` 的元素。

---

### 3. 为什么 `class` 很重要

它让你不必把所有按钮都设成一样，而可以：
- 只改某几个元素
- 给不同按钮不同样式
- 提高页面结构和样式的可维护性

这在真实项目里几乎是基础能力。

---

### 4. 不同类可以有不同样式

对应视频时间：`00:19:13`

课程演示了：
- `red-button`
- `yellow-button`

不同的类可以对应不同的 CSS 规则。

这说明：

> HTML 里用类名标记元素，CSS 再根据类名去控制外观。

## 十八、`console.log()`

### 1. 为什么需要 `console.log`

如果你把 JavaScript 写进 HTML 文件里，像：

### 2. `console.log()` 的作用

对应视频时间：`00:40:40`

课程解释说：

> `console.log()` 会把括号里的内容显示回控制台。

### 3. `console.log()` 不只能打印数字

对应视频时间：`00:41:20`

课程还演示了打印字符串表达式。  
所以它能输出很多类型的值，比如：
- 数字
- 字符串
- 后面你学到的变量、对象、数组等

---

### 4. 为什么它这么重要

补充知识点。

从这节课开始，你不再只依靠 `alert()` 看结果，而是开始使用更专业的调试方式。

`console.log()` 是后面几乎整门课程都会持续使用的基础工具。

---

## 十九、本课最重要的整体思路

课程在后半段给了一个非常关键的总结：

> 一切从 HTML 开始。  
> HTML 先创建网页，CSS 在 HTML 中改变外观，JavaScript 在 HTML 中增加交互。

这句话非常值得记住。

如果你以后做页面时思路混乱，就回到这个框架：

1. 先确定 HTML 结构
2. 再写 CSS 样式
3. 最后加 JavaScript 交互

---

## 二十、本课最容易出错的地方

### 1. HTML 标签不配对

开始标签和结束标签必须成对出现。

---

### 2. 把属性写到结束标签里

属性通常写在开始标签，不写在结束标签。

---

### 3. 忘记 `class` 选择器前面的点

CSS 里写类选择器时，应该是：

```css
.red-button
```

不是直接写类名。

---

### 4. 把 `script` 放错位置后看不懂执行时机

初学阶段最好先跟课程习惯一致，把它放在 `body` 底部。

---

### 5. `onclick` 里 JavaScript 字符串引号冲突

属性值本身常常用双引号包裹，里面再写 JavaScript 字符串时要特别注意引号搭配。

---

### 6. 以为保存文件后浏览器会自动刷新

如果没开 Live Server，通常不会自动刷新。

---

### 7. 误以为 HTML 代码里的换行一定会显示在页面上

普通 HTML 文本并不是这样工作的。

---

## 二十一、本课最重要的知识点总结

1. 完整网页由 HTML、CSS、JavaScript 三者协作构成
2. HTML 元素由开始标签、内容、结束标签组成
3. HTML 支持元素嵌套
4. HTML 中多个空格和换行通常不会原样显示
5. CSS 通过选择器选中元素，再通过属性和值改样式
6. HTML 属性可以改变元素行为或提供附加信息
7. `class` 是控制特定元素样式的核心手段之一
8. 标准页面结构包括 `<!DOCTYPE html>`、`html`、`head`、`body`
9. `script` 可以在 HTML 中运行 JavaScript
10. `onclick` 可以在点击时运行 JavaScript
11. 注释用于解释代码或临时停用代码
12. `console.log()` 是最基础的调试输出工具

---

## 二十二、建议复习顺序

1. 先重新理解 HTML、CSS、JavaScript 的分工
2. 再练习最基础的 HTML 标签和嵌套
3. 再练 CSS 选择器、属性和值
4. 再练 `class` 和类选择器
5. 再复习标准 HTML 结构
6. 最后练 `script`、`onclick`、注释和 `console.log()`

---

## 二十三、课后练习建议

1. 创建一个 HTML 文件，写一个按钮和一个段落
2. 给按钮加一个 `title` 属性
3. 用 `<style>` 把按钮背景改成你喜欢的颜色
4. 给两个按钮分别加不同的 `class`
5. 用两个不同类把两个按钮设置成不同颜色
6. 在页面底部写一个 `<script>`，用 `console.log()` 输出一段文字
7. 给一个按钮加 `onclick="alert('clicked')"`
8. 试着自己加一条 HTML 注释、一条 CSS 注释和一条 JS 注释

---

## 二十四、给初学者的最终提醒

这节课最重要的不是记住多少标签和属性，而是建立一个完整网页的心智模型：

> HTML 先把页面搭起来，CSS 负责把它变好看，JavaScript 负责让它动起来。

从这里开始，你已经不再只是“在控制台里试代码”，而是在真正做网页。

如果你把这节课的结构、`class`、`script`、`onclick` 和 `console.log()` 这些点真正掌握住，后面学 DOM 和项目交互时会顺很多。
