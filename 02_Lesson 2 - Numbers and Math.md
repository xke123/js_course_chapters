
这说明 JavaScript 能像计算器一样处理数字。

### 2. 乘法和除法的写法

对应视频时间：`00:02:16`

在 JavaScript 里：

```js

10 / 2
```

其中：
- `*` 表示乘法
- `/` 表示除法

### 4. 一次计算里可以有多个数字

对应视频时间：`00:03:03`

例如：

```js
2 + 2 + 2
```

JavaScript 会按规则把它们全部算出来。

这说明表达式 `expression` 不一定只有两个数字，也可以更长。

### 5. JavaScript 支持小数

对应视频时间：`00:03:20`

也可以正常计算。

> 小数虽然能用，但在编程里有时会带来精度问题。

这点对金额计算尤其重要。

## 五、运算顺序 Order of Operations

### 1. 什么是运算顺序

对应视频时间：`00:06:23`

课程把加减乘除称为：
- `operations`：运算
- `operators`：运算符

而“运算顺序”就是：

> 当一个表达式里有多种运算时，JavaScript 按什么顺序去计算。

---

### 2. 乘除优先于加减

对应视频时间：`00:06:59`

例如：

```js
1 + 1 * 3
```

JavaScript 会先算：

```js
1 * 3
```

再加前面的 `1`。

所以结果是：

```js
4
```

而不是 `6`。

这和普通数学规则一致。

---

### 3. 同优先级时从左到右

对应视频时间：`00:07:47`

课程提到：
- 乘法和除法优先级相同
- 加法和减法优先级相同

如果同级运算一起出现，就按从左到右的顺序计算。

例如：

```js
20 / 2 * 5
```

不是随便先算，而是从左往右。

这条规则对你以后写复杂表达式时很重要。

---

### 4. 用括号改变顺序

对应视频时间：`00:08:13`

例如：

```js
(1 + 1) * 3
```

这时括号里的内容先算，结果就变成：

```js
6
```

所以括号的作用是：

> 明确告诉 JavaScript 哪一部分必须先算。

---

### 5. 括号的语法规则

对应视频时间：`00:09:09`

课程提到两个关键要求：

1. 开括号和闭括号必须配对
2. 括号里要是完整的计算

例如：
- 少一个闭括号会报错
- 括号里写成半截表达式也不行

这既是数学问题，也是语法问题。

---

## 六、百分比的计算

### 1. JavaScript 里没有直接写 `10%` 这种语法

对应视频时间：`00:11:27`

课程强调：

> 不能直接把百分号写进普通数学表达式里做金额计算。

所以百分比要手动转成小数。

---

### 2. 百分比和小数的关系

对应视频时间：`00:11:42`

课程解释了：

```txt
1% = 1 / 100 = 0.01
10% = 10 / 100 = 0.1
```

所以如果你要计算某个金额的 `10%`，就应该乘以 `0.1`。

例如：

```js
36.93 * 0.1
```

---

### 3. 税额计算为什么容易出错

对应视频时间：`00:12:47` 到 `00:14:43`

如果你直接写：

```js
20.95 + 7.99 * 2 * 0.1
```

结果可能不符合你的业务意图，因为 JavaScript 会先乘后加。

你真正想表达的通常是：

```js
(20.95 + 7.99 * 2) * 0.1
```

也就是：
- 先算商品总价
- 再乘税率

这正是括号的重要性。

## 八、浮点数精度问题

### 1. `0.1 + 0.2` 为什么不是精确的 `0.3`

对应视频时间：`00:15:42`

课程演示了经典例子：

```js
0.1 + 0.2
```

结果不是你想象中的精确 `0.3`，而会出现一个带很多小数位的结果。

这不是你写错了，而是计算机表示浮点数时的常见问题。

---

### 2. 为什么会这样

对应视频时间：`00:16:00`

课程给了一个简化解释：

> 计算机底层用二进制存储数字，而有些十进制小数无法被精确表示。

所以某些浮点数在存储时就已经是“近似值”，后面计算自然也会有误差。

你现在不需要掌握底层细节，但一定要记住结论：

> 某些小数在编程里不精确，这是正常现象。

---

### 3. 这是不是 JavaScript 独有的问题

对应视频时间：`00:15:57`

不是。  
课程也提到，很多编程语言都会有类似问题。

所以这是“计算机处理浮点数”的共性问题，不是 JavaScript 单独设计得差。

---

## 九、金额计算的最佳实践：用分而不是美元

### 1. 为什么金额计算最好不要直接用浮点数

对应视频时间：`00:18:00`

当你算钱时，哪怕只有一点点误差，也可能带来问题。

例如：
- 显示金额不对
- 税额有偏差
- 总价与页面显示不一致

所以课程给出的最佳实践是：

> 计算金钱时，尽量先用“分”作为单位。

---

### 2. 怎么把美元改成分

对应视频时间：`00:18:12`

例如：

```txt
$20.95 -> 2095
$7.99  -> 799
```

然后你做计算时就只处理整数：

```js
2095 + 799
```

最后再除以 `100` 转回美元。

---

### 3. 这种做法的核心价值

它的好处是：
- 避开浮点数精度问题
- 让金额计算更稳定
- 更接近真实项目中的做法

这也是电商、支付、账单系统里非常常见的思路。

---

### 4. 本课中金额计算的标准思路

对应视频时间：`00:18:00` 到 `00:24:22`

可以总结成三步：

1. 先把金额转成分
2. 用整数完成计算
3. 最后除以 `100` 转回美元

如果还需要控制显示精度，再结合四舍五入处理。

## 十、`Math.round()`

### 1. `Math.round()` 的作用

对应视频时间：`00:19:49`

课程引入了：

```js
Math.round(2.2)
Math.round(2.8)
```

它的作用是：

> 把一个数字四舍五入到最接近的整数。

结果分别是：
- `2`
- `3`

---

### 2. 注意大小写

对应视频时间：`00:20:00`

课程特地提醒：

```js
Math.round()
```

里的 `Math` 首字母必须大写。

因为 JavaScript 区分大小写，写成：

```js
math.round()
```

通常就不对。

---

### 3. 为什么不能直接对美元结果四舍五入

对应视频时间：`00:22:59` 到 `00:23:39`

如果你直接对：

```js
2.894
```

使用 `Math.round()`，会得到：

```js
3
```

因为它会四舍五入到“最接近的整数”，不是保留两位小数。

所以课程的正确做法是：

> 先在“分”的层面四舍五入，再转回美元。

例如：

```js
Math.round(289.4) / 100
```

这样才会得到：

```js
2.89
```

---

### 4. 金额四舍五入的完整思路

对应视频时间：`00:24:15`

课程最后总结的流程可以概括成：

1. 先把金额计算成分
2. 对分的结果做 `Math.round()`
3. 最后再除以 `100`

这是这节课最值得记住的实战模式之一。

## 十一、嵌套括号

### 1. 括号里还可以继续有括号

对应视频时间：`00:21:52`

课程里出现了“括号里面再放括号”的情况。

JavaScript 的处理规则是：

> 先算最里面的括号，再算外面的括号。

这和普通数学是一致的。

---

### 2. 为什么这很重要

因为真实项目里的金额公式往往不会只有一层。  
可能会同时包含：
- 商品总价
- 数量
- 税率
- 折扣
- 单位转换

如果不借助括号把逻辑写清楚，表达式就很容易算错。

对应视频时间：`00:25:01`

课程给出的技巧非常实用：

这是很实际的学习方式。

## 十三、本课最容易出错的地方

### 1. 把 `*` 和 `/` 写错

初学者有时会下意识想写数学里的 `×` 或 `÷`，但 JavaScript 只能识别：
- `*`
- `/`

---

### 2. 忽略运算顺序

写出一个表达式时，如果没有想清楚“先算哪一部分”，结果就容易错。

---

### 3. 忘记用括号表达业务逻辑

你脑子里想的是“先算商品总价，再算税”，  
但如果代码里没用括号明确表达，JavaScript 只会按自己的规则算。

---

### 4. 直接用浮点数算钱

这节课最重要的坑之一就是：

```js
20.95 + 7.99
```

看起来没问题，但在某些场景下可能带来精度误差。

---

### 5. 误用 `Math.round()`

很多初学者第一次会以为：

```js
Math.round(2.894)
```

能得到 `2.89`。  
其实不会，它只会得到最接近的整数 `3`。

---

### 6. 把 `Math` 写成小写

大小写错误是入门阶段的高频问题。

---

## 十四、本课最重要的知识点总结

1. JavaScript 可以直接做数字运算
2. `*` 表示乘法，`/` 表示除法
3. JavaScript 遵循普通数学里的运算顺序
4. 括号优先级最高，可以控制计算顺序
5. 百分比要换算成小数再参与运算
6. 整数和浮点数是两类不同数字
7. 浮点数计算有时会不精确
8. 金额计算时，最好先以分为单位
9. `Math.round()` 用于四舍五入到最接近整数
10. 查资料和搜索代码是编程学习的一部分

## 十七、给初学者的最终提醒

这节课最重要的收获，不是“会算几道数学题”，而是开始理解：

从购物车金额、运费、税，到以后你会学到的折扣、库存、订单汇总，本质上都离不开这节课的基础。

如果你能真正理解运算顺序、括号、浮点误差和“用分计算金额”这几件事，后面做项目时会少踩很多坑。
