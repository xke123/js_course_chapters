## 二、本课整体理解

Lesson 3 的主题是字符串，也就是“文本”。

如果说上一课主要是在处理数字，这一课就是在解决另一个非常常见的问题：

> 我们怎样把计算结果变成用户真正能看到的文字。

例如在购物车页面中，用户看到的不是一个纯数字：

```txt
28.94
```

而是像这样一整段文本：

```txt
Items (2): $28.94
```

这说明真实网页开发里，数字和文本经常是混在一起工作的。

这一课最重要的核心不是“会写几种引号”，而是理解：

1. 字符串表示文本
2. 文本和数字是不同类型
3. 当文本和数字混合时，JavaScript 的行为会发生变化
4. 模板字符串能让这类代码更清晰

---

## 三、什么是字符串 String

### 1. 字符串的定义

对应视频时间：`00:00:41`

课程给出的定义非常直接：

> 字符串表示文本。

例如：

```js
'hello'
```

这就是一个字符串。

你可以把字符串理解成：
- 一段文字
- 一组字符
- 被 JavaScript 当作“文本值”处理的数据

---

### 2. 字符串的最基本写法

对应视频时间：`00:00:51`

创建字符串时，课程先用的是单引号：

```js
'hello'
```

规则很简单：
- 文本放中间
- 开头一个引号
- 结尾一个引号

如果没有引号，JavaScript 往往不会把它当成文本看。

---

### 3. 第一课里其实已经见过字符串

对应视频时间：`00:01:09`

课程提醒你，第一课的这段代码里就已经用到了字符串：

```js
alert('hello');
```

其中：

```js
'hello'
```

就是字符串。

这节课相当于把之前“已经用过但没正式展开讲”的概念，单独拿出来系统解释。

---

## 四、字符串的基本语法

### 1. 字符串必须被引号包住

对应视频时间：`00:01:48`

这是最基础的语法规则。

例如：

```js
'hello'
```

是字符串，而：

```js
hello
```

通常不是字符串，JavaScript 会把它理解成别的东西，比如变量名。

---

### 2. 为什么引号这么重要

因为引号在告诉 JavaScript：

> 这里面的内容不是命令，不是变量，而是普通文本。

这和数字不同：

```js
2
```

本身就是数字值，不需要额外符号。  
但文本必须靠引号标记出来。

---

## 五、字符串拼接 Concatenation

### 1. 字符串可以用 `+` 连接

对应视频时间：`00:02:01`

例如：

```js
'some' + 'text'
```

结果会变成一个更大的字符串。

---

### 2. 什么是拼接 `concatenation`

对应视频时间：`00:02:57`

当我们把多个字符串连接成一个字符串，这就叫拼接。

例如：

```js
'some' + 'more' + 'text'
```

课程里强调，这就是把几段文本加在一起。

注意：

> 对字符串来说，`+` 不是“数学加法”，而是“把文本接起来”。

---

### 3. 拼接时空格要自己处理

补充知识点。

很多初学者第一次会写：

```js
'hello' + 'world'
```

结果得到：

```txt
helloworld
```

因为字符串拼接不会自动帮你补空格。  
如果想要空格，通常得自己写进去：

```js
'hello' + ' ' + 'world'
```

这在后面拼接页面文案时非常常见。

---

## 六、数字和字符串是不同的数据类型

### 1. 这是两种不同的值类型

对应视频时间：`00:03:04`

课程明确说：
- 数字 `number`
- 字符串 `string`

是 JavaScript 中两种不同的值类型。

它们分别表示不同东西：
- 数字表示数量或数值
- 字符串表示文本

这是后面理解类型强制的基础。

---

### 2. 用 `typeof` 检查类型

对应视频时间：`00:03:15`

你可以用：

```js
typeof 2
typeof 'hello'
```

来查看某个值的类型。

结果会分别是：
- `number`
- `string`

---

### 3. `typeof` 的意义

补充知识点。

`typeof` 很适合初学阶段用来确认：
- 当前这个值到底是不是字符串
- 我到底是在处理数字还是文本

很多时候代码结果不符合预期，就是因为你以为它是数字，其实它已经变成字符串了。

---

## 七、字符串和数字相加：类型强制

### 1. 字符串和数字相加会发生什么

对应视频时间：`00:03:48`

例如：

```js
'hello' + 3
```

结果不是报错，也不是数学加法。  
JavaScript 会把数字 `3` 自动变成字符串，再拼接起来。

结果类似：

```txt
hello3
```

---

### 2. 什么是类型强制 `type coercion`

对应视频时间：`00:04:23`

课程把这种自动转换叫做：

> `type coercion`

也就是：

> JavaScript 在某些情况下，会自动把一个值转换成另一种类型。

这里的例子就是：
- 数字 `3`
- 自动变成字符串 `'3'`

---

### 3. 为什么这是高频坑点

因为很多初学者看到 `+` 就默认以为在做加法。  
但如果表达式里一旦出现字符串，`+` 的行为就可能变成拼接。

这也是本课项目例子里最关键的问题来源。

---

## 八、用字符串和数字生成项目中的文本

### 1. 为什么项目里不能只显示数字

对应视频时间：`00:04:33`

页面上真正给用户看的不是裸数字，而是类似：

```txt
$28.94
Items (2): $28.94
```

这些都是“文本 + 数字”的组合结果。

所以这节课开始练习：

> 先算出数字，再把数字放进字符串里。

---

### 2. 直接把结果硬编码成字符串当然可以，但不够好

对应视频时间：`00:05:00`

例如你当然可以直接写：

```js
'$28.94'
```

但课程强调，JavaScript 的优势是能计算。

所以更好的思路是：
- 不要死写结果
- 让代码自己算出金额
- 再把结果放进文本里

这才更接近真实项目。

---

## 九、为什么字符串和数字混用时结果会错

### 1. 典型错误示例

对应视频时间：`00:06:09`

例如：

```js
'$' + 20.95 + 7.99
```

初学者可能以为结果会是：

```txt
$28.94
```

但实际并不是这样。

---

### 2. 为什么会这样

对应视频时间：`00:06:37`

因为 JavaScript 从左到右计算：

1. 先算：

```js
'$' + 20.95
```

这一步是“字符串 + 数字”，所以会变成字符串拼接。

2. 得到字符串后，再加上 `7.99`，就又会继续拼接。

于是整个表达式不再是数学计算，而变成了文本连接。

---

### 3. 解决方法：先算数字，再拼接字符串

对应视频时间：`00:07:36`

正确思路是：

```js
'$' + (20.95 + 7.99)
```

括号会强制 JavaScript 先做数学计算，得到数字结果后，再和字符串拼接。

这说明：

> 字符串相关代码同样会受到运算顺序影响。

---

## 十、字符串也遵循运算顺序

### 1. 这不是只有数字才有的规则

对应视频时间：`00:07:13`

课程特别强调：

> 字符串和数字混合时，也要考虑运算顺序。

也就是说：
- 括号依然优先
- 乘除依然先于加减
- 从左到右的规则依然有效

只是这时 `+` 的含义可能变成拼接，不一定是加法。

---

### 2. 这节课最值得真正理解的一个点

补充知识点。

很多人学到这里，会把问题归结成“字符串不好用”。  
其实不是字符串的问题，而是你必须学会判断：

> 当前这个 `+` 到底是在做数字加法，还是字符串拼接。

这是一条非常关键的思维分界线。

---

## 十一、用字符串和数字拼整行文本

### 1. 课程中的整行文本示例

对应视频时间：`00:09:45`

课程开始拼像这样的整行内容：

```txt
Items (2): $28.94
```

这时字符串构建会更长，通常需要：
- 一部分固定文本
- 一部分动态数量
- 一部分固定符号
- 一部分动态价格

---

### 2. 串联式写法的思路

课程本质上是在做：

1. 把整行文本拆成多个部分
2. 对数字部分单独计算
3. 最后把所有部分拼回去

这是早期 JavaScript 项目里非常经典的做法。

---

### 3. 创建好的字符串可以交给其他代码使用

对应视频时间：`00:12:29`

例如课程里把拼好的字符串放到：

```js
alert(...)
```

里显示出来。

这说明字符串一旦创建完成，就可以被其他 JavaScript 代码继续使用：
- 弹窗
- 输出到页面
- 保存到变量
- 再次拼接

---

## 十二、三种创建字符串的方法

### 1. 单引号

对应视频时间：`00:13:31`

```js
'hello'
```

这是课程前半段最常用的方式。

---

### 2. 双引号

对应视频时间：`00:13:56`

```js
"hello"
```

它和单引号创建的是同一种字符串。

课程建议默认优先使用单引号，因为更常见、也更顺手。

---

### 3. 反引号 / 模板字符串

对应视频时间：`00:18:35`

```js
`hello`
```

这也是字符串，但它有额外功能，所以有专门名字：

> 模板字符串 `template strings`

---

## 十三、什么时候用双引号

### 1. 当字符串内部本身有单引号

对应视频时间：`00:14:27`

例如：

```js
"I'm learning JavaScript"
```

这里如果你用单引号包外层，字符串内部的 `'` 可能会把字符串提前结束。

所以改用双引号是很自然的解决办法。

---

### 2. 这不是必须规则，而是实用习惯

补充知识点。

你并不是“只能这样写”，而是：

> 在这个场景下，双引号更方便。

后面你也会学到另一种解决办法：转义字符。

---

## 十四、转义字符 Escape Characters

### 1. 什么是转义字符

对应视频时间：`00:15:33`

课程解释说，转义字符是一组特殊字符，通常以反斜杠 `\` 开头。

它们的作用是：

> 在字符串里表示一些原本有特殊含义，或者不方便直接写的内容。

---

### 2. `\\'` 和 `\\\"`

对应视频时间：`00:16:26`

例如：

```js
'I\'m learning JavaScript'
```

这里的：

```js
\'
```

表示一个普通的单引号字符，它不会结束字符串。

类似地：

```js
\"
```

可以表示普通双引号字符。

---

### 3. `\n`：换行字符

对应视频时间：`00:17:28`

课程还讲了：

```js
'\n'
```

它不表示字母 `n`，而表示“新的一行”。

例如：

```js
'some\ntext'
```

会让文本换到下一行。

---

### 4. 为什么转义字符重要

因为它让你能在字符串里安全表达：
- 引号
- 换行
- 其他特殊内容

这在弹窗、日志、HTML 字符串、模板文本中都很常见。

---

## 十五、模板字符串 Template Strings

### 1. 模板字符串是什么

对应视频时间：`00:18:35`

模板字符串是用反引号创建的字符串：

```js
`hello`
```

它和普通字符串一样能表示文本，但它还有两个很重要的额外功能：
- 插值 `interpolation`
- 多行字符串 `multi-line strings`

---

## 十六、插值 Interpolation

### 1. 什么是插值

对应视频时间：`00:19:58`

插值允许你把一个值直接放进字符串里。

写法是：

```js
`${...}`
```

例如：

```js
`Items (${1 + 1})`
```

这会先计算：

```js
1 + 1
```

再把结果放进字符串里。

---

### 2. 为什么插值比拼接更推荐

对应视频时间：`00:20:19`

课程比较了两种写法后，明确指出：

> 插值通常更简洁，也更清楚。

例如之前用拼接要写很多：
- 引号
- 加号
- 括号

而模板字符串可以把整个文本写得更接近最终显示效果。

---

### 3. 插值里不只能放变量，也能放表达式

补充知识点。

`$ {...}` 里面不一定只能放单个值，也可以直接写计算：

```js
`${2095 + 799}`
```

或者：

```js
`${(2095 + 799) / 100}`
```

这正是它比传统拼接方便的地方。

---

## 十七、多行字符串 Multi-line Strings

### 1. 模板字符串可以直接换行

对应视频时间：`00:21:56`

例如：

```js
`some
text`
```

这本身就是合法的字符串。

---

### 2. 它和 `\n` 的关系

课程提到，这种效果本质上和换行字符类似：
- 都能产生新的一行
- 只是模板字符串写起来更自然

如果文本本来就是多行内容，模板字符串通常会更好读。

---

## 十八、单引号、双引号、模板字符串怎么选

### 1. 课程建议

对应视频时间：`00:22:45`

课程给出的建议很实用：

- 默认用单引号
- 如果字符串里有单引号，可以考虑双引号
- 如果要插入值或写多行文本，用模板字符串

这是一套非常适合初学者的选择规则。

---

### 2. 实际开发中的理解

补充知识点。

重点不是“哪一种最高级”，而是：

> 哪一种最清晰、最不容易出错。

通常：
- 简单纯文本：单引号就够
- 含引号冲突：双引号或转义字符
- 动态文本：模板字符串更舒服

---

## 十九、本课最容易出错的地方

### 1. 把字符串和数字相加当成数学加法

这是本课最大的坑。

---

### 2. 忘记加括号

如果你想先算数字，再拼接文本，通常就需要用括号控制顺序。

---

### 3. 拼接时忘了空格

字符串不会自动帮你补空格。

---

### 4. 引号配对错误

字符串开头和结尾的引号必须成对。

---

### 5. 字符串内部出现同种引号导致提前结束

例如：

```js
'I'm learning'
```

就可能出问题。

---

### 6. 把反引号和单引号混淆

模板字符串用的是反引号 `` ` ``，不是普通单引号 `'`。

---

10. 模板字符串支持多行文本

7. 用模板字符串生成一句类似 `Items (2): $28.94` 的文本

## 二十三、给初学者的最终提醒

这节课最重要的不是“记住三种引号”，而是建立这种能力：

> 我不仅能让 JavaScript 算出结果，还能把结果变成用户真正看得懂的文本。

从这节课开始，你已经在接触真实前端开发里非常常见的一类工作：
- 计算数据
- 组织文字
- 把动态值插入界面文本

如果你把“字符串、拼接、类型强制、模板字符串”这几件事真的学明白，后面做 DOM、页面渲染和项目拼接时会轻松很多。
