## 三、函数是什么

### 1. 函数的本质

对应视频时间：`00:01:50`

函数本质上是一段“可以重复使用的代码”。

你可以把它理解为：

1. 先给一段代码起个名字
2. 以后需要时，用这个名字把代码运行起来

例如：

```js
function greet() {
  console.log('hello');
}
```

这里的 `greet` 就是函数名。

---

### 2. 为什么需要函数

对应视频时间：`00:04:01`、`00:11:20`

函数主要解决两个问题：

1. 减少重复代码
2. 让代码更容易修改和维护

如果你在三个地方都写了相同逻辑，以后只要改规则，你就得改三次。  
如果这段逻辑抽成函数，你通常只需要改一处。

这就是“封装”的最早期体验。

---

## 四、如何创建函数和调用函数

### 1. 创建函数

对应视频时间：`00:02:01`

基本语法：

```js
function functionName() {
  // code
}
```

例如：

```js
function greet() {
  console.log('hello');
  console.log('welcome');
}
```

这叫做“定义函数”或“创建函数”。

---

### 2. 调用函数

对应视频时间：`00:03:04`、`00:05:09`

调用函数就是让函数执行：

```js
greet();
```

注意这里必须有括号 `()`。  
只有写函数名而不加括号，通常表示“指向这个函数”，不是执行它。

---

### 3. 定义函数不等于执行函数

对应视频时间：`00:05:03`

这是初学者最容易混淆的一点。

```js
function greet() {
  console.log('hello');
}
```

这段代码只是把函数创建出来，并不会自动打印 `hello`。

只有当你写：

```js
greet();
```

它才真正执行。

你可以把它理解成：

1. 定义函数：把工具造出来
2. 调用函数：真正使用这个工具

---

### 4. 函数体 `function body`

对应视频时间：`00:04:46`

函数花括号 `{}` 里面的代码，叫函数体。

```js
function greet() {
  console.log('hello');
  console.log('welcome');
}
```

这里两行 `console.log` 都属于函数体。

函数体里的代码，只有在函数被调用时才会运行。

---

## 五、函数命名规则

### 1. 命名要表达动作

对应视频时间：`00:07:13`

函数通常表示“做一件事”，所以名字最好是动词或动作短语：

1. `pickComputerMove`
2. `playGame`
3. `calculateTax`
4. `showMessage`

不推荐太模糊的名字，比如：

1. `data`
2. `thing`
3. `abc`

这些名字无法体现函数的用途。

---

### 2. 使用驼峰命名 `camelCase`

对应视频时间：`00:26:47`

JavaScript 里函数名通常使用 `camelCase`：

```js
pickComputerMove
calculateTax
updateCartQuantity
```

规则是：

1. 第一个单词小写
2. 后面的单词首字母大写

---

## 六、函数如何帮助我们重构重复代码

### 1. 石头剪刀布中的重复逻辑

对应视频时间：`00:05:24`、`00:06:26`

课程里先发现：每个按钮里都有一段很像的“电脑随机出拳”逻辑。

如果在三个按钮中都写一遍：

1. 代码重复
2. 容易改漏
3. 可读性差

所以应该把这段逻辑抽成一个函数，比如：

```js
function pickComputerMove() {
  // 随机生成电脑出拳
}
```

这样每次需要电脑出拳时，只要调用：

```js
pickComputerMove();
```

---

### 2. 函数的价值不只是“少写几行”

函数更重要的价值是：把一个明确的逻辑块抽象出来。

比如你看到：

```js
const computerMove = pickComputerMove();
```

即使不看内部实现，你也大概知道它在做什么。  
这会让代码更容易读。

---

## 七、函数作用域 scope

### 1. 什么是作用域

对应视频时间：`00:09:08`

作用域可以简单理解为：

一个变量在什么范围内可以被访问。

函数会创建自己的作用域。

例如：

```js
function test() {
  const message = 'hello';
  console.log(message);
}

test();
console.log(message);
```

最后一行会报错，因为 `message` 只存在于 `test` 函数内部。

---

### 2. 函数内部变量，外部不能直接访问

这是非常重要的规则：

1. 函数里面定义的变量，通常只能在函数里面使用
2. 函数外面不能直接读到它

这可以防止不同代码块互相干扰。

---

### 3. 全局变量和局部变量

课程里先展示了一种“把变量移到函数外”的办法。

对应视频时间：`00:10:06`

例如：

```js
let computerMove = '';

function pickComputerMove() {
  computerMove = 'rock';
}
```

这里 `computerMove` 是定义在函数外部的变量，叫全局变量 `global variable`。

而函数内部自己声明的变量，通常叫局部变量 `local variable`。

---

### 4. 为什么过度依赖全局变量不好

对应视频时间：`00:19:22`

全局变量虽然有时方便，但问题很多：

1. 容易被其他地方误改
2. 不容易追踪是谁修改了值
3. 函数对外部环境依赖太强

所以课程后面给出了更好的方案：用 `return` 返回结果，而不是依赖全局变量。

---

## 八、返回值 return

### 1. `return` 是什么

对应视频时间：`00:12:20`

`return` 的作用是：

把函数内部的值“返回”到函数外面。

例如：

```js
function getNumber() {
  return 5;
}
```

调用：

```js
console.log(getNumber());
```

会输出：

```js
5
```

---

### 2. `return` 的方向

你可以把它理解成：

1. 参数：把值传进函数
2. `return`：把值从函数里传出来

这两个概念是一对。

字幕里也明确讲了这个对比。

对应视频时间：`00:22:26`

---

### 3. 用 `return` 改写石头剪刀布逻辑

对应视频时间：`00:16:00`、`00:17:15`

比起把 `computerMove` 放到全局，更推荐这样：

```js
function pickComputerMove() {
  const randomNumber = Math.random();

  if (randomNumber < 1 / 3) {
    return 'rock';
  } else if (randomNumber < 2 / 3) {
    return 'paper';
  } else {
    return 'scissors';
  }
}

const computerMove = pickComputerMove();
```

这样写的优点：

1. 函数更独立
2. 函数更容易复用
3. 函数更容易测试

---

### 4. 没有 `return` 会得到什么

对应视频时间：`00:14:47`

如果函数没有写 `return`，默认返回：

```js
undefined
```

例如：

```js
function test() {
  console.log('hello');
}

console.log(test());
```

输出顺序会是：

```js
hello
undefined
```

原因是：

1. 函数执行时打印了 `hello`
2. 但它没有返回明确值
3. 所以调用结果是 `undefined`

---

### 5. `return` 会立刻结束函数

对应视频时间：`00:15:00`

这是 `return` 很重要的行为。

```js
function test() {
  console.log('a');
  return 5;
  console.log('b');
}
```

这里 `console.log('b')` 永远不会执行。  
因为函数在执行到 `return 5;` 时已经结束了。

所以：

1. `return` 不只是“返回值”
2. 它还会“提前结束函数”

---

### 6. `return` 后面可以放什么

理论上，`return` 后面可以是任何表达式：

```js
return 5;
return 'hello';
return true;
return 1 + 2;
return randomNumber;
```

只要这个表达式最终能算出一个值。

---

## 九、如何追踪函数执行过程

### 1. 代码追踪的重要性

对应视频时间：`00:18:28`、`00:39:10`

这一课反复做了一件事：逐行追踪代码到底是怎么执行的。

这是初学者非常需要培养的能力。

看代码时建议按这个顺序：

1. 哪一行先执行？
2. 这个函数有没有被调用？
3. 参数传了什么值？
4. 返回了什么值？
5. 变量最后变成了什么？

---

### 2. 学函数时不要只记语法

真正决定你会不会函数的，不是你能不能背出：

```js
function abc() {}
```

而是你能不能在脑子里模拟：

1. 什么时候进入函数
2. 什么时候退出函数
3. 哪个值流进去了
4. 哪个值流出来了

这也是后面学调试 `debugging` 的基础。

## 十、参数 parameters

### 1. 参数是什么

对应视频时间：`00:21:31`、`00:22:26`

参数的作用是：让函数从外部接收数据。

例如：

```js
function greet(name) {
  console.log(`Hello ${name}`);
}
```

这里的 `name` 就是参数。

调用时：

```js
greet('Alice');
greet('Bob');
```

同一个函数就可以处理不同数据。

---

### 2. 参数的语法

对应视频时间：`00:24:35`

参数写在函数名后面的括号里：

```js
function calculateTax(cost) {
  console.log(cost * 0.1);
}
```

这里的 `cost` 是参数。

---

### 3. 传入参数

对应视频时间：`00:24:50`

调用函数时，把值写进括号里：

```js
calculateTax(2000);
calculateTax(5000);
```

这里的 `2000`、`5000` 会分别传给参数 `cost`。

---

### 4. 参数在函数内部像变量一样使用

对应视频时间：`00:25:30`

参数本质上就是函数内部可用的一个变量名：

```js
function calculateTax(cost) {
  console.log(cost * 0.1);
}
```

这里 `cost` 可以参与运算、拼接字符串、判断条件等。

---

### 5. 参数只在函数内部有效

对应视频时间：`00:28:30`

参数和普通局部变量一样，也只存在于函数作用域内。

```js
function calculateTax(cost) {
  console.log(cost);
}

calculateTax(2000);
console.log(cost);
```

最后一行会报错，因为 `cost` 不在函数外部可用。

---

## 十一、参数和实参 argument

### 1. 两个术语的区别

对应视频时间：`00:27:25`

严格来说：

1. `parameter` 参数：函数定义时括号里的名字
2. `argument` 实参：函数调用时实际传入的值

例如：

```js
function greet(name) {
  console.log(name);
}

greet('Alice');
```

这里：

1. `name` 是参数 `parameter`
2. `'Alice'` 是实参 `argument`

---

### 2. 初学阶段怎么理解最实用

初学时你可以先抓住这个核心：

1. 定义函数时写的是“接收位”
2. 调用函数时写的是“传入值”

术语要知道，但不要被术语卡住。

---

## 十二、多个参数

### 1. 多个参数的写法

对应视频时间：`00:28:46`、`00:29:14`

多个参数用逗号分隔：

```js
function calculateTax(cost, taxPercent) {
  console.log(cost * taxPercent);
}
```

调用：

```js
calculateTax(2000, 0.1);
calculateTax(5000, 0.2);
```

---

### 2. 位置必须对应

多个参数传值时，顺序很重要：

```js
function calculateTax(cost, taxPercent) {
  console.log(cost * taxPercent);
}
```

如果你调用：

```js
calculateTax(0.1, 2000);
```

那就会把：

1. `cost = 0.1`
2. `taxPercent = 2000`

结果就错了。

所以参数传值默认是“按位置匹配”的。

---

## 十三、默认参数 default parameters

### 1. 没传值时会发生什么

对应视频时间：`00:29:58`

如果少传了一个参数，那么对应参数通常会得到：

```js
undefined
```

例如：

```js
function calculateTax(cost, taxPercent) {
  console.log(cost * taxPercent);
}

calculateTax(2000);
```

此时 `taxPercent` 是 `undefined`。

---

### 2. 默认参数的语法

对应视频时间：`00:30:36`

可以给参数设置默认值：

```js
function calculateTax(cost, taxPercent = 0.1) {
  console.log(cost * taxPercent);
}
```

这样如果调用：

```js
calculateTax(2000);
```

就相当于：

```js
taxPercent = 0.1
```

---

### 3. 默认参数的意义

默认参数适合这种情况：

1. 某个值通常有默认规则
2. 但有时也允许外部覆盖

例如：

```js
calculateTax(2000);
calculateTax(2000, 0.2);
```

第一种用默认税率，第二种手动指定税率。

## 十四、用参数重构石头剪刀布项目

### 1. 进一步消除重复

对应视频时间：`00:31:06`

前面虽然已经有了 `pickComputerMove()`，但三个按钮里仍然有大量重复逻辑。

课程进一步把“玩家出拳 + 电脑出拳 + 判断结果 + 显示结果”统一进：

```js
function playGame(playerMove) {
  // ...
}
```

这样三个按钮只需要传不同的参数：

```js
playGame('rock');
playGame('paper');
playGame('scissors');
```

---

### 2. `playerMove` 参数的意义

对应视频时间：`00:32:50`、`00:33:00`

`playerMove` 表示玩家当前选择的出拳。

调用时不同，函数内部逻辑就会用到不同值：

```js
playGame('scissors');
```

这意味着：

1. 这次函数执行里，`playerMove` 是 `'scissors'`
2. 下次调用时，它又可能变成 `'rock'` 或 `'paper'`

这正是参数让函数更灵活的地方。

---

### 3. 在函数内部使用参数做判断

对应视频时间：`00:33:36`、`00:35:48`、`00:37:26`

例如：

```js
if (playerMove === 'scissors') {
  // 判断剪刀对电脑出拳的结果
} else if (playerMove === 'paper') {
  // 判断布
} else if (playerMove === 'rock') {
  // 判断石头
}
```

这说明参数不仅能传入数据，还能控制函数内部走哪条逻辑分支。

---

### 4. 用模板字符串显示结果

对应视频时间：`00:34:31`

在生成提示文本时，会把参数插入字符串：

```js
`You picked ${playerMove}. Computer picked ${computerMove}.`
```

这里顺便强化了上一课的模板字符串知识。

## 十五、函数可以调用其他函数

### 1. 这是正常且常见的写法

对应视频时间：`00:38:45`

例如：

```js
function pickComputerMove() {
  return 'rock';
}

function playGame(playerMove) {
  const computerMove = pickComputerMove();
  console.log(playerMove, computerMove);
}
```

这里 `playGame` 内部调用了 `pickComputerMove`。

这种写法非常常见。

---

### 2. 为什么要拆成多个函数

拆函数的好处是：

1. 每个函数职责更单一
2. 更容易复用
3. 更容易调试
4. 更容易测试

例如：

1. `pickComputerMove()` 只负责生成电脑出拳
2. `playGame(playerMove)` 负责一整轮游戏逻辑

职责清晰后，代码会更稳定。

---

## 十六、脚本顺序与函数可用性

### 1. 为什么按钮能调用函数

对应视频时间：`00:08:03`

课程里提到，脚本先执行，把函数创建出来；之后用户点击按钮时，按钮中的 `onclick` 才会调用这些函数。

这说明你要理解：

1. 页面加载时，`<script>` 中的代码会先运行
2. 函数先被创建
3. 用户后续交互时，相关函数已经存在了

---

### 2. 初学阶段应该掌握的实际结论

如果你在 HTML 中点击按钮要调用某个函数，那么通常需要保证：

1. 这个函数已经在脚本里定义好了
2. 脚本确实被页面加载了

否则点击时会报“函数未定义”之类的错误。

---

## 十七、函数写法中的几个基础补充

### 1. 函数也是组织代码的工具

即使某段代码不重复，有时你也可以把它写成函数。  
因为函数还能让代码更清晰，便于按逻辑分块。

例如：

1. 读取用户输入
2. 计算结果
3. 更新页面

这些都可以各自封装成函数。

---

### 2. 不要让函数做太多事情

初学时容易写出特别长的函数，把很多不相关逻辑都塞进去。

更好的习惯是：

1. 一个函数尽量只做一件主要事情
2. 名字要能概括它的任务

这会让后面的重构更轻松。

---

### 3. 函数不一定非要返回值

有些函数的目标是“产生副作用”，比如：

1. `console.log(...)`
2. `alert(...)`
3. 修改页面内容

这种函数可以不返回值。  
但如果你想把“计算结果”传出来，就应该考虑 `return`。

---

## 十八、常见错误与易混点

### 1. 只定义函数，没有调用

错误思路：

```js
function greet() {
  console.log('hello');
}
```

然后以为它已经执行了。实际上没有。

---

### 2. 忘记加括号

```js
greet;
```

这通常不是调用函数，而只是写出了函数名。

正确写法：

```js
greet();
```

---

### 3. 在函数外访问函数内部变量

```js
function test() {
  const message = 'hello';
}

console.log(message);
```

会报错，因为作用域不对。

---

### 4. 把 `return` 和 `console.log` 混为一谈

`console.log` 是打印到控制台。  
`return` 是把值返回给函数调用处。

它们不是一回事。

例如：

```js
function test() {
  console.log(5);
}
```

这会打印 `5`，但函数返回的仍然是 `undefined`。

---

### 5. 参数名和传入值混淆

```js
function greet(name) {
  console.log(name);
}
```

这里 `name` 只是一个接收位置，不是固定值。  
真正的值是在调用时传进去的。

---

### 6. 漏传参数却没考虑默认值

```js
function calculateTax(cost, taxPercent) {
  return cost * taxPercent;
}

calculateTax(2000);
```

这里结果通常不是你想要的。  
应该考虑默认参数：

```js
function calculateTax(cost, taxPercent = 0.1) {
  return cost * taxPercent;
}
```

---

## 十九、本课最重要的结论

这节课最需要你真正掌握的，不是零散语法，而是下面这些核心结论：

1. 函数是可以复用的一段代码
2. 定义函数不等于执行函数
3. 调用函数要写 `()`
4. 函数会形成自己的作用域
5. 要把结果从函数里带出来，用 `return`
6. 要让函数接收外部数据，用参数
7. 参数和返回值的方向相反
8. 多个参数按位置匹配
9. 漏传参数时，没默认值通常会得到 `undefined`
10. 一段重复逻辑，应该优先考虑抽成函数

## 二十、建议复习顺序

如果你准备回看这一课，建议按这个顺序复习：

1. 先理解函数是什么，为什么需要它
2. 再练“定义函数”和“调用函数”
3. 再理解函数作用域
4. 再理解 `return`
5. 再理解参数和默认参数
6. 最后回看石头剪刀布的重构过程

这样顺序最稳，因为后面的内容都建立在前面的基础上。

## 二十一、练习建议

### 1. 基础练习

1. 写一个函数，打印 `hello`
2. 写一个函数，返回数字 `10`
3. 写一个函数，接收一个名字参数并打印欢迎语
4. 写一个函数，接收两个数字并返回它们的和

### 2. 参数练习

1. 写一个 `calculateTax(cost, taxPercent = 0.1)` 函数
2. 分别传一个参数和两个参数，观察结果
3. 尝试不传参数，看是否得到 `undefined`

### 3. 作用域练习

1. 在函数内部定义变量并打印
2. 再尝试在函数外打印同名变量
3. 观察报错并解释原因

### 4. 项目练习

1. 把石头剪刀布中重复的逻辑继续抽成更小函数
2. 写一个 `getResult(playerMove, computerMove)` 函数，只负责返回比赛结果
3. 写一个 `showResult(...)` 函数，只负责显示结果文本

这样你会更快理解“一个函数只做一件事”的思路。

## 二十二、你接下来应该具备的能力

学完这课后，你应该开始具备这些能力：

1. 能主动发现重复代码
2. 能把重复逻辑提取成函数
3. 能判断一个值应该作为参数传入，还是作为返回值传出
4. 能初步看懂由多个函数组成的小程序

如果这些还不稳，不要急着往后跳。  
因为后面的数组、对象、DOM、模块、测试，都会反复建立在函数之上。
