## 三、数组 Array

### 1. 数组是什么

对应视频时间：`00:02:33`

数组是 JavaScript 中的一种值，用来表示一组有顺序的数据。

```js
[10, 20, 30]
```

这个数组里有 3 个值：
- `10`
- `20`
- `30`

可以把数组理解成：

> 一个按顺序排列的值列表。

---

### 2. 为什么需要数组

对应视频时间：`00:02:33`

如果你只保存一个值，用变量就够了：

```js
const score = 100;
```

但如果你要保存很多相关的值，比如一组待办事项：

```js
const todo1 = 'make dinner';
const todo2 = 'wash dishes';
const todo3 = 'study JavaScript';
```

这样写会越来越乱。

更好的做法是：

```js
const todoList = ['make dinner', 'wash dishes', 'study JavaScript'];
```

数组的意义就是：
- 把多个相关值放在一起
- 方便统一处理
- 方便循环遍历

---

### 3. 如何创建数组

对应视频时间：`00:02:42`

语法：

```js
const myArray = [10, 20, 30];
```

规则：
- 用方括号 `[]`
- 元素之间用逗号 `,` 分隔
- 数组本身也是一个值，所以可以赋给变量

空数组：

```js
const todoList = [];
```

这在项目里很常见，表示“现在先没有任何数据，后面再往里加”。

---

### 4. 数组中的元素可以是什么

数组里可以放很多种类型：

```js
const arr = [1, 'hello', true];
```

也可以放对象、数组、函数：

```js
const arr = [
  1,
  { name: 'Alice' },
  [10, 20]
];
```

JavaScript 不强制数组里必须是同一种类型。

但是补充一个开发建议：

> 虽然 JS 允许混合类型，但实际开发中，通常让一个数组保存同一类数据会更清晰。

比如：
- 一个数字数组
- 一个字符串数组
- 一个对象数组

这样更容易维护。

---

### 5. 数组索引 index

对应视频时间：`00:04:12`

数组中的每个元素都有位置，这个位置叫索引 `index`。

```js
const myArray = [10, 20, 30];
```

它们的索引分别是：
- `10` 的索引是 `0`
- `20` 的索引是 `1`
- `30` 的索引是 `2`

注意：

> 数组索引从 `0` 开始，不是从 `1` 开始。

这叫做：
- zero-based index
- 零基索引

这是初学者最容易犯错的点之一。

---

### 6. 读取数组中的值

对应视频时间：`00:04:12`

通过方括号和索引读取：

```js
const myArray = [10, 20, 30];

console.log(myArray[0]); // 10
console.log(myArray[1]); // 20
console.log(myArray[2]); // 30
```

语法结构：

```js
数组名[索引]
```

例如：

```js
todoList[0]
```

表示“取出数组中的第一个元素”。

---

### 7. 修改数组中的值

对应视频时间：`00:05:23`

数组里的某个位置可以直接重新赋值：

```js
const myArray = [10, 20, 30];
myArray[0] = 99;

console.log(myArray); // [99, 20, 30]
```

这里要理解一个容易混淆的点：

```js
const myArray = [10, 20, 30];
```

即使使用的是 `const`，你依然可以修改数组内部元素。

为什么？

因为：
- `const` 限制的是变量不能重新指向别的值
- 但数组内部内容仍然可以改

不能这样：

```js
const myArray = [10, 20, 30];
myArray = [1, 2, 3]; // 报错
```

但可以这样：

```js
myArray[0] = 1;
```

---

### 8. 数组的 `length`

对应视频时间：`00:09:36`

`length` 表示数组中元素的数量。

```js
const myArray = [10, 20, 30];
console.log(myArray.length); // 3
```

重要关系：

```js
最后一个索引 = 数组长度 - 1
```

例如：

```js
const arr = [10, 20, 30];
console.log(arr.length); // 3
console.log(arr[2]);     // 最后一个元素
```

为什么是 `2`？

因为索引从 `0` 开始：
- `0`
- `1`
- `2`

所以长度是 3，但最后一个索引是 2。

---

### 9. `typeof` 和数组

对应视频时间：`00:08:18`

字幕里提到一个“奇怪现象”：

```js
const arr = [1, 2, 3];
console.log(typeof arr); // object
```

为什么不是 `array`？

因为 JavaScript 历史上把数组归到对象类型里。

所以：
- `typeof []` 返回的是 `"object"`
- 这不能准确判断它是不是数组

正确判断数组的方法是：

```js
Array.isArray(arr); // true
```

这是实际开发中更标准的写法。

---

### 10. `push()` 向数组末尾添加元素

对应视频时间：`00:10:06`

```js
const todoList = [];
todoList.push('make dinner');

console.log(todoList); // ['make dinner']
```

作用：

> 把一个值加到数组最后面。

这是本课 Todo List 项目里最重要的方法之一。

再看一个例子：

```js
const nums = [1, 2];
nums.push(3);

console.log(nums); // [1, 2, 3]
```

补充：
- `push()` 会修改原数组
- 它通常返回新长度，但这节课更重要的是理解“它会把元素加到最后”

---

### 11. `splice()` 删除数组中的元素

对应视频时间：`00:10:42`

本课用它来删除待办事项。

基本写法：

```js
const arr = [10, 20, 30];
arr.splice(0, 1);

console.log(arr); // [20, 30]
```

含义：
- 第一个参数：从哪个索引开始删
- 第二个参数：删几个

再看两个例子：

```js
const arr = [10, 20, 30, 40];
arr.splice(1, 2);

console.log(arr); // [10, 40]
```

这表示：
- 从索引 `1` 开始
- 删除两个元素，也就是 `20` 和 `30`

补充理解：
- `splice()` 会直接修改原数组
- 它很常用于“按位置删除”

---

### 12. `slice()` 复制数组

对应视频时间：`01:33:09`

字幕在讲“数组是引用类型”时提到了 `slice()`。

```js
const array1 = [1, 2, 3];
const array2 = array1.slice();
```

这会创建一个新数组，复制原数组中的元素。

它和 `splice()` 不一样：
- `slice()` 主要用于复制或截取，不改原数组
- `splice()` 主要用于删除或插入，会改原数组

这是初学者很容易混淆的一组方法。

---

## 四、数组是引用类型

### 1. 什么叫引用

对应视频时间：`01:31:24`

```js
const array1 = [1, 2, 3];
const array2 = array1;
```

很多初学者以为 `array2` 是 `array1` 的一份独立复制。

其实不是。

真正发生的是：
- `array1` 指向一块数组数据
- `array2` 也指向同一块数组数据

所以：

```js
array2.push(4);

console.log(array1); // [1, 2, 3, 4]
console.log(array2); // [1, 2, 3, 4]
```

两个变量都会看到变化。

---

### 2. 为什么这很重要

因为你以后写项目时，数组经常会在：
- 函数之间传递
- 变量之间赋值
- 对象属性之间共享

如果你不知道数组是引用类型，就会出现：
- 不知道是谁改了数据
- 明明改的是 `array2`，结果 `array1` 也变了

---

### 3. 如何复制数组

对应视频时间：`01:33:09`

本课字幕提到的方法是：

```js
const array2 = array1.slice();
```

补充一个现在更常见的写法：

```js
const array2 = [...array1];
```

这叫展开运算符 `spread syntax`，你后面课程可能会学到。

但在当前阶段，先记住：

> `const copy = original.slice();` 可以复制一个新数组。

---

### 4. 浅拷贝补充

这里顺便补一个会影响理解的点：

`slice()` 复制的是“第一层元素”，叫浅拷贝。

如果数组里放的是对象：

```js
const arr1 = [{ name: 'A' }];
const arr2 = arr1.slice();

arr2[0].name = 'B';
console.log(arr1[0].name); // B
```

这是因为：
- 外层数组复制了
- 里面的对象还是同一个对象

这一点你现在不用完全吃透，但要有概念：复制数组不一定等于复制数组内部所有东西。

## 五、循环 Loop

### 1. 循环是什么

对应视频时间：`00:24:48`

循环的本质是：

> 让一段代码重复执行。

比如：
- 连续打印 1 到 10
- 遍历数组里的每个元素
- 为数组中的每个待办项生成 HTML

---

### 2. 为什么需要循环

如果没有循环，你可能要手写：

```js
console.log(todoList[0]);
console.log(todoList[1]);
console.log(todoList[2]);
```

这只适用于元素数量固定且很少的情况。

如果数组里有 100 个元素，就不现实了。

所以循环的作用是：
- 自动重复
- 节省代码
- 适应不同数量的数据

---

## 六、while 循环

### 1. 基本语法

对应视频时间：`00:25:19`

```js
let i = 1;

while (i <= 5) {
  console.log(i);
  i++;
}
```

执行过程：
- 先看 `i <= 5` 是否为真
- 如果为真，就执行大括号里的代码
- 执行完后更新 `i`
- 再回去检查条件
- 条件为假时停止

---

### 2. while 循环的三个核心部分

对应视频时间：`00:29:19`

一个正常的 while 循环通常有三部分：

#### 1. 循环变量

```js
let i = 1;
```

#### 2. 循环条件

```js
i <= 5
```

#### 3. 更新步骤

```js
i++
```

字幕里把这三部分讲得很明确，这对初学者非常重要。

---

### 3. 迭代 iteration

对应视频时间：`00:30:44`

循环每执行一次，叫一次迭代 `iteration`。

例如：

```js
let i = 1;
while (i <= 3) {
  console.log(i);
  i++;
}
```

这个循环一共执行了 3 次迭代。

---

### 4. 死循环 infinite loop

对应视频时间：`00:29:47`

如果循环条件一直为真，循环就不会结束，这叫死循环。

比如：

```js
let i = 1;

while (i <= 5) {
  console.log(i);
}
```

这里没有 `i++`，所以 `i` 永远是 `1`。

结果：
- 条件一直成立
- 程序一直跑

这是本课必须非常警惕的错误。

---

## 七、for 循环

### 1. 基本语法

对应视频时间：`00:31:35`

```js
for (let i = 1; i <= 5; i++) {
  console.log(i);
}
```

它和下面的 while 效果相同：

```js
let i = 1;

while (i <= 5) {
  console.log(i);
  i++;
}
```

---

### 2. for 循环的三部分

```js
for (初始化; 条件; 更新) {
  // 循环体
}
```

例如：

```js
for (let i = 0; i < 3; i++) {
  console.log(i);
}
```

含义：
- 初始化：`let i = 0`
- 条件：`i < 3`
- 更新：`i++`

---

### 3. 为什么 for 更常用于数组遍历

对应视频时间：`00:33:35`

字幕里提到：
- 标准循环更适合用 `for`
- 非标准循环更适合用 `while`

这句话非常有价值。

什么叫标准循环？

就是这种你很清楚循环结构的情况：
- 从 0 开始
- 到某个条件结束
- 每次固定加 1

数组遍历就是最典型的标准循环，所以最常用 `for`。

---

### 4. for 和 while 的选择

对应视频时间：`00:33:35`、`00:35:51`

#### 用 `for` 的场景
- 遍历数组
- 打印固定范围数字
- 已知循环变量怎么变化

#### 用 `while` 的场景
- 什么时候停不确定
- 条件更灵活
- 例如“不断生成随机数，直到满足某条件”

---

## 八、非标准循环：随机数示例

对应视频时间：`00:34:06`

字幕里有这样一个思路：

> 一直生成随机数，直到随机数大于等于 0.5。

例如：

```js
let randomNumber = 0;

while (randomNumber < 0.5) {
  randomNumber = Math.random();
}

console.log(randomNumber);
```

为什么这里更适合 `while`？

因为：
- 你不知道要执行几次
- 结束条件依赖随机结果

这就是一个很典型的“非标准循环”。

## 九、遍历数组

### 1. 什么叫遍历

对应视频时间：`00:36:18`

遍历数组，就是把数组中的每个元素依次拿出来处理。

例如：

```js
const todoList = ['make dinner', 'wash dishes', 'study'];
```

你想把每一项都打印出来，就需要遍历。

---

### 2. 最常见写法

对应视频时间：`00:37:16`

```js
const todoList = ['make dinner', 'wash dishes', 'study'];

for (let i = 0; i < todoList.length; i++) {
  console.log(todoList[i]);
}
```

这里：
- `i` 是索引
- `todoList[i]` 是当前索引对应的元素

---

### 3. 为什么条件是 `i < arr.length`

对应视频时间：`00:38:03`

这是字幕里强调的重点之一。

如果数组长度是 3：

```js
const arr = ['a', 'b', 'c'];
```

合法索引只有：
- `0`
- `1`
- `2`

所以循环应写成：

```js
i < arr.length
```

而不是：

```js
i <= arr.length
```

因为 `arr.length` 等于 `3`，`arr[3]` 不存在。

补充：

```js
console.log(arr[3]); // undefined
```

所以 `i < arr.length` 是最标准也最安全的写法。

---

### 4. `index` 与 `i`

字幕里既用过 `index` 也用过 `i`。

这两个本质没区别，只是变量名不同：

```js
for (let index = 0; index < arr.length; index++) {
  console.log(arr[index]);
}
```

和

```js
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}
```

都可以。

建议：
- 学习阶段用 `i` 最常见
- 当你特别想强调“这是索引”时，用 `index` 可读性更高

## 十、累加器模式 Accumulator Pattern

### 1. 什么是累加器模式

对应视频时间：`00:41:29`

字幕把它作为本课重要技巧单独讲了。

累加器模式的核心思路是：

1. 先创建一个变量存结果
2. 再循环遍历数据
3. 每一轮更新结果变量

这个“存结果的变量”就叫累加器。

---

### 2. 例子：求数组总和

对应视频时间：`00:41:54`

```js
const nums = [1, 1, 3];
let total = 0;

for (let i = 0; i < nums.length; i++) {
  const num = nums[i];
  total += num;
}

console.log(total); // 5
```

分析：
- `total` 是结果变量
- 一开始是 `0`
- 每轮循环把当前数字加进去
- 最终得到总和

---

### 3. 为什么必须用 `let`

如果你要不断修改结果：

```js
let total = 0;
```

就必须用 `let`。

不能用：

```js
const total = 0;
```

因为：

```js
total += num;
```

这相当于重新赋值，`const` 不允许。

这个点字幕里也提到了。

---

### 4. 例子：生成一个翻倍后的新数组

对应视频时间：`00:45:06`

```js
const nums = [1, 1, 3];
const numsDoubled = [];

for (let i = 0; i < nums.length; i++) {
  const num = nums[i];
  numsDoubled.push(num * 2);
}

console.log(numsDoubled); // [2, 2, 6]
```

这里累加器不再是数字，而是数组。

说明：

> 累加器不一定只能是数字，也可以是数组、字符串，甚至对象。

---

### 5. 初学者对累加器模式的正确理解

不要把它死记为某个固定模板，而要理解它的本质：

> 结果不是一次得到的，而是在循环过程中一边遍历一边累积出来的。

这会在以后大量出现，例如：
- 求和
- 统计数量
- 过滤结果
- 生成 HTML
- 构造新数组

## 十一、Todo List 项目中的数组与循环

对应视频时间：基础版本 `00:18:05`，渲染版 `00:48:48`

这一课的项目价值很高，因为它把“数组 + 循环 + DOM”连起来了。

---

### 1. 第一步：准备一个数组存待办事项

```js
const todoList = [];
```

这个数组就是页面数据的来源。

注意一个重要思想：

> 页面显示的内容，最好由数据驱动。

也就是说，不是先写死页面，而是先准备数据，再让页面根据数据渲染。

---

### 2. 第二步：获取输入框内容

字幕中用的是：

```js
const inputElement = document.querySelector('.js-name-input');
const name = inputElement.value;
```

这里复习了上一课 DOM 的知识：
- `querySelector()` 获取元素
- `value` 获取输入框内容

---

### 3. 第三步：把用户输入加入数组

```js
todoList.push(name);
```

每次点击“Add”按钮，就把当前输入内容塞进数组末尾。

这样数组就会从：

```js
[]
```

变成：

```js
['make dinner']
```

再变成：

```js
['make dinner', 'wash dishes']
```

---

### 4. 第四步：清空输入框

```js
inputElement.value = '';
```

这一步虽然不复杂，但很重要，因为它说明：

> `value` 不只是可以读取，也可以修改。

也就是说：
- 读取：拿到输入内容
- 写入：修改输入框显示内容

---

### 5. 第五步：把数组内容显示到页面

对应视频时间：`00:49:09`

这里开始进入本课最重要的部分：

> 用循环根据数组动态生成页面内容。

一个常见做法是：

```js
let todoListHTML = '';

for (let i = 0; i < todoList.length; i++) {
  const todo = todoList[i];
  todoListHTML += `<p>${todo}</p>`;
}

document.querySelector('.js-todo-list').innerHTML = todoListHTML;
```

这里其实就是把“累加器模式”换了一个形式：
- 结果变量不再是数字
- 变成了字符串 `todoListHTML`

每次循环都往这个字符串里拼一点 HTML。

---

### 6. 重新渲染 render 的思想

对应视频时间：`00:57:05`

字幕里把渲染逻辑提取成函数，例如 `renderTodoList()`。

这种思路非常好：

```js
function renderTodoList() {
  let todoListHTML = '';

  for (let i = 0; i < todoList.length; i++) {
    const todo = todoList[i];
    todoListHTML += `<p>${todo}</p>`;
  }

  document.querySelector('.js-todo-list').innerHTML = todoListHTML;
}
```

好处：
- 页面更新逻辑集中在一个地方
- 新增、删除后只要重新调用这个函数
- 代码更整洁

核心思想可以总结成一句话：

> 先修改数据，再重新渲染页面。

这是前端开发非常重要的基础观念。

## 十二、删除待办项

对应视频时间：`01:01:09` 到 `01:05:35`

### 1. 为什么删除要知道索引

数组删除时，本课用的是：

```js
todoList.splice(i, 1);
```

这里必须知道：
- 删的是第几个元素

而在循环里刚好有索引 `i`，所以可以直接用。

---

### 2. 删除按钮的思路

当循环生成每一条 todo 的 HTML 时，也顺便生成对应的删除按钮。

思路是：
- 第 0 项的删除按钮删第 0 项
- 第 1 项的删除按钮删第 1 项
- 第 2 项的删除按钮删第 2 项

字幕里是把 `i` 插进生成的 HTML 字符串中。

这本质上是在做：

> 把“当前循环位置”绑定到当前按钮上。

---

### 3. 删除后为什么还要重新渲染

删除数据只是改了数组：

```js
todoList.splice(i, 1);
```

但页面不会自动更新，所以还要：

```js
renderTodoList();
```

这进一步强化了本课的核心思路：

1. 改数组
2. 再把数组重新显示到页面

## 十三、把字符串数组升级为对象数组

对应视频时间：`01:06:40`

### 1. 为什么字符串不够用

一开始每个 todo 只是一个名字：

```js
['make dinner', 'wash dishes']
```

但后面新增了截止日期 `due date`。

此时每个 todo 不只是一段文字，而是包含两份信息：
- 名称
- 日期

这时候字符串就不够用了。

---

### 2. 用对象表示一个待办项

```js
{
  name: 'make dinner',
  dueDate: '2022-12-22'
}
```

这样每一项都是一个对象。

整个数组就变成：

```js
const todoList = [
  {
    name: 'make dinner',
    dueDate: '2022-12-22'
  },
  {
    name: 'wash dishes',
    dueDate: '2022-12-23'
  }
];
```

这就是对象数组。

---

### 3. 为什么对象数组很常见

因为真实项目里，一条数据往往不止一个字段。

例如：
- 用户：姓名、年龄、邮箱
- 商品：名称、价格、库存
- 待办项：名称、日期、状态

所以：

> 数组负责保存一组数据，对象负责描述单条数据的多个属性。

这个组合非常常见。

---

### 4. 遍历对象数组时怎么取值

当数组中每个元素是对象时：

```js
const todoObject = todoList[i];
const name = todoObject.name;
const dueDate = todoObject.dueDate;
```

然后再把这两个值插入到页面中。

这一步实际上把前面学过的“对象”和这节课的“数组、循环”串了起来。

## 十四、解构 Destructuring

对应视频时间：对象解构 `01:09:02`，数组解构 `01:33:46`

字幕先讲了对象解构，后面也讲了数组解构。

---

### 1. 对象解构

普通写法：

```js
const name = todoObject.name;
const dueDate = todoObject.dueDate;
```

解构写法：

```js
const { name, dueDate } = todoObject;
```

这会直接把对象中同名属性取出来放进变量里。

好处：
- 更简洁
- 更清晰
- 减少重复书写对象名

---

### 2. 数组解构

例如：

```js
const array = [1, 2, 3];
const [firstValue, secondValue] = array;
```

等价于：

```js
const firstValue = array[0];
const secondValue = array[1];
```

这个语法后面你会越来越常见。

---

### 3. 初学者对解构的理解方式

可以先把它理解成：

> 一种更短的“取值并存到变量”的写法。

不必一开始就纠结所有高级用法，先掌握最基础的两个：
- 对象解构
- 数组解构

## 十五、`break` 和 `continue`

对应视频时间：`01:34:54`

这两个是循环控制语句。

---

### 1. `break`：提前结束整个循环

```js
for (let i = 1; i <= 10; i++) {
  if (i === 8) {
    break;
  }

  console.log(i);
}
```

效果：
- 打印到 `7`
- 当 `i === 8` 时直接结束整个循环

可以理解成：

> 不继续了，整个循环到此为止。

---

### 2. `continue`：跳过本次循环

```js
for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    continue;
  }

  console.log(i);
}
```

输出：

```js
1
2
4
5
```

因为当 `i === 3` 时：
- 本轮剩余代码不执行
- 直接进入下一轮

可以理解成：

> 这一次不做，继续下一次。

---

### 3. 用 `%` 判断是否整除

字幕里结合 `continue` 讲了 `%` 取余运算符。

```js
if (i % 3 === 0) {
  continue;
}
```

含义：
- `i % 3` 表示 `i` 除以 `3` 的余数
- 如果余数为 `0`，说明能被 `3` 整除

例如：
- `3 % 3 === 0`
- `6 % 3 === 0`
- `7 % 3 === 1`

所以可以用来跳过所有能被 3 整除的数字。

---

### 4. `continue` 在 `while` 中的陷阱

对应视频时间：`01:38:04`

这是本课一个非常容易考、也非常容易错的点。

看这个例子：

```js
let i = 1;

while (i <= 10) {
  if (i % 3 === 0) {
    continue;
  }

  console.log(i);
  i++;
}
```

这里有问题。

当 `i === 3` 时：
- 进入 `if`
- 执行 `continue`
- 直接跳到下一轮
- `i++` 被跳过了

结果：
- `i` 一直是 `3`
- 条件一直成立
- 出现死循环

正确写法是：

```js
let i = 1;

while (i <= 10) {
  if (i % 3 === 0) {
    i++;
    continue;
  }

  console.log(i);
  i++;
}
```

所以要牢牢记住：

> 在 `while` 循环里使用 `continue` 时，要特别小心循环变量是否还能正常更新。

---

## 十六、循环和函数结合使用

对应视频时间：`01:40:05`

### 1. 为什么要把循环写进函数里

如果某段循环逻辑以后还会重复使用，就应该封装成函数。

例如：把数组中的每个数字都翻倍。

如果直接写死在代码里，只能处理一个数组。

更好的方式是：

```js
function doubleArray(nums) {
  const numsDoubled = [];

  for (let i = 0; i < nums.length; i++) {
    const num = nums[i];
    numsDoubled.push(num * 2);
  }

  return numsDoubled;
}
```

然后：

```js
console.log(doubleArray([1, 1, 3]));
console.log(doubleArray([2, 2, 5]));
```

这样同一个循环逻辑就可以反复复用。

---

### 2. 参数让循环更通用

函数里的参数：

```js
function doubleArray(nums) {
```

意味着：
- 这个函数不依赖某个写死的数组
- 任何传进来的数组都可以处理

这一步非常关键，因为它体现了：

> 函数的价值在于把通用逻辑封装起来。

---

### 3. `return` 不只是返回值，也会结束函数

字幕里提到在函数内部，可以用 `return` 提前停止循环。

例如：

```js
function doubleArray(nums) {
  const numsDoubled = [];

  for (let i = 0; i < nums.length; i++) {
    const num = nums[i];

    if (num === 0) {
      return numsDoubled;
    }

    numsDoubled.push(num * 2);
  }

  return numsDoubled;
}
```

这里当遇到 `0` 时：
- `return` 会立刻结束整个函数
- 后面的循环也不会再继续

所以在函数内部：
- `break` 结束循环
- `return` 结束整个函数

这两者要区分清楚。

## 十七、这一课隐含但非常重要的前端思想

这一部分字幕没有系统总结，但学习时应该主动补上。

### 1. 数据和界面分开

项目中：
- `todoList` 是数据
- 页面 HTML 是显示结果

正确顺序是：

1. 先维护数据
2. 再根据数据生成界面

而不是反过来靠手工改 HTML 维护状态。

### 2. 页面不是“存数据”的地方

初学者经常会想：

> 页面上已经显示了 todo，为什么还要数组？

因为页面只是显示结果，不是最可靠的数据来源。

真正应该信任的是数组：

```js
const todoList = [...]
```

界面只是它的可视化结果。

---

### 3. 渲染是“从数据重新生成界面”

这一课的 `renderTodoList()` 本质是在做：

> 给我一份最新数据，我重新画一遍页面。

这和后面很多框架的思路非常像。

你现在学的虽然是原生 JS，但思想上已经接近：
- React 的 render

所以这节课是很重要的过渡课。

## 十八、字幕提到但值得补充的相关知识点

### 1. 模板字符串 template literal

字幕中在生成 HTML 时，实际上已经在使用模板字符串：

'<p>' + todo + '</p>'

模板字符串更易读。

element.innerHTML = todoListHTML;

含义：

在当前学习阶段先会用就行，后面再理解更深入的安全问题。

## 十九、本课常见错误

### 1. 把数组索引当成从 1 开始

错误思路：
- 第一个元素是 `arr[1]`

正确：
- 第一个元素是 `arr[0]`

---

### 2. 遍历数组时写成 `i <= arr.length`

错误：

```js
for (let i = 0; i <= arr.length; i++) {
  console.log(arr[i]);
}
```

最后一次会拿到 `undefined`。

正确：

```js
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}
```

---

### 3. 忘记在 while 中更新循环变量

这会导致死循环。

---

### 4. 在 while 中用 `continue` 却忘记手动递增

这也是死循环高发点。

---

### 5. 把 `array2 = array1` 当成真正复制

这只是复制引用，不是复制内容。

---

### 6. 修改了数组，但忘了重新渲染页面

比如：
- `push()` 了新数据
- `splice()` 删除了旧数据

但如果不重新执行 `renderTodoList()`，页面显示不会更新。

---

### 7. 把对象数组还按字符串数组去处理

升级为：

```js
[{ name: 'a', dueDate: '2022-12-22' }]
```

之后，就不能再直接把某项当字符串插入了，要先取属性：

```js
todoObject.name
todoObject.dueDate
```

## 二十二、复习版总结

这一课最核心的内容可以压缩成下面几句话：

- 数组是按顺序保存多个值的数据结构。
- 数组通过索引访问，索引从 `0` 开始。
- `length` 表示数组长度，最后一个索引是 `length - 1`。
- `push()` 用来在数组末尾添加元素，`splice()` 用来按索引删除元素。
- 循环用于重复执行代码，最常见的是 `while` 和 `for`。
- 遍历数组最常见的写法是：

```js
for (let i = 0; i < arr.length; i++) {
  const value = arr[i];
}
```

- 累加器模式是“先准备结果变量，再在循环中不断更新结果”。
- Todo List 的本质是：先维护数组，再根据数组重新生成页面。
- 当一条数据包含多个属性时，应该使用对象数组。
- 数组是引用类型，直接赋值不会复制内容。
- `break` 结束整个循环，`continue` 跳过本次循环。
