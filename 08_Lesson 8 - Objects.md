## 三、对象是什么

### 1. 对象的本质

对应视频时间：`00:01:46`

对象是 JavaScript 里另一种类型的值。  
它的作用是把多个相关的值组合在一起。

例如：

```js
const product = {
  name: 'socks',
  price: 1090
};
```

这里 `product` 不是单个值，而是一组相关信息：

1. 商品名 `name`
2. 商品价格 `price`

---

### 2. 为什么要用对象

对应视频时间：`00:08:28`

如果一个事物有很多相关信息，用对象通常比很多零散变量更清晰。

比如：

```js
const name = 'socks';
const price = 1090;
const deliveryTime = '1 day';
```

这些值彼此有关，但分散在不同变量里。

改成对象之后：

```js
const product = {
  name: 'socks',
  price: 1090,
  deliveryTime: '1 day'
};
```

好处是：

1. 结构更清晰
2. 数据更集中
3. 更适合后面扩展

## 四、如何创建对象

### 1. 基本语法

对应视频时间：`00:02:00`、`00:05:35`

对象使用花括号 `{}` 创建：

```js
const product = {
  name: 'socks',
  price: 1090
};
```

这里每一项都叫“属性-值对”：

1. `name: 'socks'`
2. `price: 1090`

---

### 2. 属性 `property`

对应视频时间：`00:03:19`

对象里左边这部分叫属性：

```js
name: 'socks'
```

其中：

1. `name` 是属性名
2. `'socks'` 是属性值

属性名的作用是帮助我们访问对象里的具体值。

---

### 3. 对象是一种值

对应视频时间：`00:08:00`

对象本身也是 JavaScript 的值，所以：

1. 可以保存进变量
2. 可以传给函数
3. 可以作为函数返回值
4. 可以被 `console.log`

```js
const product = {
  name: 'socks'
};

console.log(product);
console.log(typeof product); // object
```

---

## 五、点记法 `dot notation`

### 1. 读取对象属性

对应视频时间：`00:03:38`

读取属性最常见的写法是点记法：

```js
console.log(product.name);
console.log(product.price);
```

含义是：

1. 先找到对象 `product`
2. 再取出它的某个属性值

---

### 2. 修改属性值

对应视频时间：`00:04:24`

对象的属性值可以直接改：

```js
product.name = 'cotton socks';
```

然后：

```js
console.log(product.name); // cotton socks
```

---

### 3. 添加新属性

对应视频时间：`00:06:31`

如果属性原本不存在，也可以直接新增：

```js
product.isNew = true;
```

这会给对象多加一个属性。

---

### 4. 删除属性

对应视频时间：`00:07:25`

可以用 `delete` 删除对象属性：

```js
delete product.isNew;
```

注意：

1. 删除的是属性
2. 不是删除整个对象变量

---

### 5. 访问不存在的属性

对应视频时间：`00:06:18`

如果访问一个不存在的属性，结果通常是：

```js
undefined
```

例如：

```js
console.log(product.notExist); // undefined
```

这和数组访问不存在索引很像。

## 六、用对象保存石头剪刀布分数

### 1. 为什么分数适合放进对象

对应视频时间：`00:09:14`、`00:13:01`

石头剪刀布的分数包含三项：

1. 赢了几次 `wins`
2. 输了几次 `losses`
3. 平了几次 `ties`

这三项明显属于同一组数据，所以很适合放进一个对象：

```js
const score = {
  wins: 0,
  losses: 0,
  ties: 0
};
```

---

### 2. 为什么 `score` 要定义在函数外面

对应视频时间：`00:13:43`

课程里特别解释了，`score` 不能定义在每次点击都会重新执行的函数内部。

因为如果写在函数里：

1. 每次运行函数都会重新创建新对象
2. 分数就会被重置

所以要把 `score` 放在函数外，让它保存“上一次”的状态。

这其实就是“状态 state”的最早期概念。

---

### 3. 根据结果更新对象属性

对应视频时间：`00:14:39` 到 `00:17:10`

例如：

```js
if (result === 'You win') {
  score.wins += 1;
} else if (result === 'You lose') {
  score.losses += 1;
} else if (result === 'Tie') {
  score.ties += 1;
}
```

这里就是根据结果修改对象中的不同属性。

---

### 4. 用模板字符串显示分数

对应视频时间：`00:17:20`

课程里把分数插入到了模板字符串里：

```js
alert(`Wins: ${score.wins}, Losses: ${score.losses}, Ties: ${score.ties}`);
```

这说明对象属性可以和模板字符串直接配合使用。

---

### 5. 多行模板字符串的空格问题

对应视频时间：`00:19:12`

如果你在模板字符串里换行并缩进：

```js
`line1
  line2`
```

这些空格也会进入最终字符串。  
所以课程里专门把多余缩进去掉了。

这是一个很常见的初学者细节坑。

---

### 6. 重置分数按钮

对应视频时间：`00:19:47` 到 `00:21:33`

重置分数的做法很直接，就是把每个属性重新设为 `0`：

```js
score.wins = 0;
score.losses = 0;
score.ties = 0;
```

这也再次说明：对象里的每个属性都可以单独读写。

---

## 七、方括号记法 `bracket notation`

### 1. 基本写法

对应视频时间：`00:22:42`

除了点记法，还可以用方括号记法访问属性：

```js
console.log(product['name']);
```

它和：

```js
console.log(product.name);
```

效果相同。

---

### 2. 什么时候必须用方括号记法

对应视频时间：`00:24:24`

如果属性名不适合点记法，比如带有 `-`：

```js
product['delivery-time']
```

不能写成：

```js
product.delivery-time
```

因为 JavaScript 会把中间的 `-` 理解成减号。

---

### 3. 方括号里可以放变量和表达式

对应视频时间：`00:26:00`

这是方括号记法的重要优势：

```js
const propertyName = 'name';
console.log(product[propertyName]);
```

也就是说，方括号里不一定写死字符串，也可以放：

1. 变量
2. 计算结果
3. 表达式

而点记法做不到这一点。

---

### 4. 点记法和方括号记法怎么选

对应视频时间：`00:26:13`

通常默认优先用点记法，因为：

1. 更短
2. 更直观
3. 更好读

只有在下面情况才优先考虑方括号记法：

1. 属性名带特殊字符
2. 属性名是动态的
3. 你需要用变量计算属性名

---

## 八、对象里可以放什么值

### 1. 可以放任意类型的值

对应视频时间：`00:26:38`

对象里不只可以放字符串和数字，还能放：

1. 布尔值
2. 对象
3. 函数

这意味着对象是非常灵活的数据结构。

---

### 2. 嵌套对象 `nested objects`

对应视频时间：`00:27:02`

对象里还可以再放对象：

```js
const product = {
  name: 'shirt',
  rating: {
    stars: 4.5,
    count: 87
  }
};
```

这里 `rating` 又是一个对象。

这样做的好处是把更细的一组相关数据再组织起来。

---

### 3. 访问嵌套对象属性

对应视频时间：`00:27:59`

访问嵌套属性时，继续连着写：

```js
console.log(product.rating.count);
```

这表示：

1. 先拿到 `product`
2. 再拿到其中的 `rating`
3. 再拿到 `rating` 里的 `count`

## 九、方法 `method`

### 1. 对象里可以保存函数

对应视频时间：`00:29:24`

例如：

```js
const product = {
  fun: function function1() {
    console.log('function inside object');
  }
};
```

这里 `fun` 这个属性对应的值是一个函数。

---

### 2. 什么是方法

对应视频时间：`00:30:35`

当一个函数保存为对象的属性时，这个函数通常叫“方法” `method`。

所以：

1. 普通函数：单独存在
2. 方法：属于某个对象的函数

调用方法时：

```js
product.fun();
```

---

### 3. 你之前已经在用方法了

对应视频时间：`00:30:50`、`00:31:20`

课程里特别指出：

1. `console.log()` 其实就是方法
2. `Math.random()` 其实也是方法

因为：

1. `console` 是对象
2. `log` 是保存在 `console` 对象里的函数

同理：

1. `Math` 是对象
2. `random` 是保存在 `Math` 对象里的函数

## 十、内建对象 `built-in objects`

### 1. 什么是内建对象

对应视频时间：`00:31:40`

内建对象就是 JavaScript 语言本身就提供好的对象，比如：

1. `console`
2. `Math`
3. `JSON`
4. `localStorage`

这些对象不是你自己创建的，而是浏览器或 JavaScript 环境直接提供的。

---

### 2. 为什么要认识内建对象

因为你以后写 JavaScript 时，不会只创建自己的对象，也会大量使用内建对象提供的能力，比如：

1. 打印日志
2. 生成随机数
3. 处理 JSON
4. 持久化保存数据

## 十一、JSON

### 1. `JSON` 是什么

对应视频时间：`00:32:40`

`JSON` 全称是 `JavaScript Object Notation`。  
它看起来很像 JavaScript 对象语法，但更严格、更通用。

你可以把它理解为：

一种用来表示数据的通用文本格式。

---

### 2. JSON 和对象语法的区别

对应视频时间：`00:33:00`

课程里强调了两个主要区别：

1. `JSON` 里的属性名和字符串都必须用双引号 `"`
2. `JSON` 不支持函数

例如：

```json
{
  "name": "shirt",
  "price": 799
}
```

---

### 3. 为什么要用 JSON

对应视频时间：`00:33:48`

因为普通 JavaScript 对象语法只在 JavaScript 里有意义。  
而 `JSON` 更通用，几乎所有编程语言都能理解。

所以它特别适合：

1. 发送数据给别的计算机
2. 跨语言传递数据
3. 存储结构化数据

---

### 4. `JSON.stringify()`

对应视频时间：`00:34:24`

这个方法可以把 JavaScript 对象转成 JSON 字符串：

```js
const jsonString = JSON.stringify(product);
```

要点：

1. 返回值是字符串
2. 不是对象
3. 函数属性不会保留下来

---

### 5. `JSON.parse()`

对应视频时间：`00:36:10`

这个方法可以把 JSON 字符串再转回 JavaScript 对象：

```js
const obj = JSON.parse(jsonString);
```

所以：

1. `stringify`：对象 -> 字符串
2. `parse`：字符串 -> 对象

这两个方法通常成对出现。

---

## 十二、`localStorage`

### 1. 为什么需要 `localStorage`

对应视频时间：`00:37:42`

变量只存在于当前页面运行期间。  
一旦刷新页面，变量里的数据通常就没了。

而 `localStorage` 可以把数据保存得更久一些。  
刷新页面后，它通常还在。

这就是它和普通变量最大的区别。

---

### 2. `setItem()`

对应视频时间：`00:39:45`

保存数据到 `localStorage`：

```js
localStorage.setItem('message', 'hello');
```

这里：

1. 第一个参数是名字 `key`
2. 第二个参数是要保存的值

---

### 3. `getItem()`

对应视频时间：`00:40:50`

从 `localStorage` 取出数据：

```js
localStorage.getItem('message');
```

如果之前保存过，就能取回来。

---

### 4. `localStorage` 只保存字符串

对应视频时间：`00:42:12`

这是本课最重要的技术细节之一。

`localStorage` 不能直接保存对象，所以保存对象前要先转成字符串：

```js
localStorage.setItem('score', JSON.stringify(score));
```

读取回来后再转回对象：

```js
const score = JSON.parse(localStorage.getItem('score'));
```

---

### 5. 在项目中保存分数

对应视频时间：`00:42:12` 到 `00:45:48`

完整思路是：

1. 页面加载时，从 `localStorage` 读取之前保存的分数
2. 如果有，就恢复它
3. 每次游戏后更新分数对象
4. 更新后再把新的分数对象保存回 `localStorage`

这就是一个最基础的数据持久化流程。

---

### 6. `removeItem()`

对应视频时间：`00:46:12`

删除 `localStorage` 中保存的值：

```js
localStorage.removeItem('score');
```

课程里在“重置分数”按钮中加了这一步，避免刷新后又读回旧分数。

## 十三、`null` 和默认值问题

### 1. `getItem()` 取不到值时会得到什么

对应视频时间：`00:47:24`

如果 `localStorage` 里没有某个键，`getItem()` 会返回：

```js
null
```

而不是空对象，也不是空字符串。

---

### 2. 为什么会报错

对应视频时间：`00:47:40`

如果你以为读到的是对象，继续访问属性：

```js
score.wins
```

但其实 `score` 是 `null`，就会报错。  
因为不能在 `null` 上读取属性。

---

### 3. 解决办法：给默认对象

对应视频时间：`00:48:00`

一种写法是先判断：

```js
if (score === null) {
  score = {
    wins: 0,
    losses: 0,
    ties: 0
  };
}
```

这就是给“读不到数据”的情况提供默认值。

---

### 4. `null` 和 `undefined` 的区别

对应视频时间：`00:52:20`

两者都表示“没有值”，但常见理解是：

1. `undefined`：通常表示“没有提供”或“没有返回”
2. `null`：通常表示“明确地设置为空”

课程里也提到，默认参数通常只会在参数是 `undefined` 时生效，不会在参数是 `null` 时生效。

---

### 5. 使用 `if (!score)`

对应视频时间：`00:49:14`

因为 `null` 是 `falsy`，所以：

```js
if (!score) {
  // 使用默认值
}
```

这是 `score === null` 的一种简写思路。

---

### 6. 默认运算符 `||`

对应视频时间：`00:50:20`

更进一步，可以直接写成：

```js
let score = JSON.parse(localStorage.getItem('score')) || {
  wins: 0,
  losses: 0,
  ties: 0
};
```

含义是：

1. 左边如果有有效值，就用左边
2. 左边如果是 `null` 这种 `falsy` 值，就用右边默认对象

这是这课里非常实用的写法。

---

## 十四、自动装箱 `auto-boxing`

### 1. 为什么字符串也能有属性和方法

对应视频时间：`00:52:56` 到 `00:55:00`

例如：

```js
'hello'.length
'hello'.toUpperCase()
```

字符串明明不是对象，但却能像对象一样用属性和方法。

这是因为 JavaScript 有自动装箱机制。

---

### 2. 自动装箱的含义

对应视频时间：`00:56:00`

JavaScript 会在某些情况下，临时把字符串、数字、布尔值包装成特殊对象，让你能调用它们的属性和方法。

例如：

1. 字符串可以用 `.length`
2. 字符串可以用 `.toUpperCase()`

---

### 3. 哪些值不会自动装箱

对应视频时间：`00:56:35`

课程里特别说了：

1. `null`
2. `undefined`

这两个值不会自动装箱。  
所以你对它们访问属性或方法时，通常会直接报错。

---

## 十五、对象是引用类型 `reference`

### 1. 对象变量里保存的不是“整个对象本体”

对应视频时间：`00:56:17`

课程里用了一个很重要的概念：对象是引用。

你可以先这样理解：

1. 真正的对象内容存在内存某处
2. 变量里保存的是“指向它的引用”

这和数字、字符串那种直接保存值的感觉不一样。

---

### 2. 对象赋值不会复制整个对象

对应视频时间：`00:57:40`

```js
const object1 = { message: 'hello' };
const object2 = object1;
```

这里通常不是“复制出一个新对象”，而是：

1. `object1` 指向原对象
2. `object2` 也指向同一个原对象

---

### 3. 改一个，另一个也会看到变化

对应视频时间：`00:58:34` 到 `01:00:03`

```js
object1.message = 'good job';
console.log(object2.message); // good job
```

原因不是 `object2` 自动同步了，  
而是它们本来就指向同一个对象。

---

### 4. 为什么 `const` 对对象内部修改无效

对应视频时间：`00:58:55`

`const` 的意思是：

1. 变量本身不能重新指向别处

但它并不阻止你修改对象内部属性：

```js
const obj = { message: 'hello' };
obj.message = 'good job'; // 可以
```

不能做的是：

```js
obj = {};
```

---

### 5. 比较对象时比的是引用，不是内容

对应视频时间：`01:00:25` 到 `01:02:16`

```js
const object1 = { message: 'good job' };
const object3 = { message: 'good job' };

console.log(object1 === object3); // false
```

虽然内容一样，但它们是两个不同对象、两个不同引用。

而：

```js
const object2 = object1;
console.log(object2 === object1); // true
```

这是因为它们指向同一个对象。

---

### 6. 如果要比较对象内容怎么办

课程里给出的实际思路是：手动比较属性。

例如：

```js
object1.message === object3.message
```

初学阶段先理解这一层就够了：

1. 对象 `===` 比较的是引用
2. 不是自动深度比较内部所有值

---

## 十六、解构 `destructuring`

### 1. 为什么要用解构

对应视频时间：`01:02:38`

如果你想从对象里取出属性，并保存成同名变量，普通写法是：

```js
const message = object4.message;
```

解构可以简写成：

```js
const { message } = object4;
```

---

### 2. 多个属性一起解构

对应视频时间：`01:04:38`

```js
const { message, price } = object4;
```

这相当于：

```js
const message = object4.message;
const price = object4.price;
```

---

### 3. 解构的本质

它不是魔法，本质上只是：

1. 从对象中取出对应属性
2. 放到同名变量里

它让代码更短，但前提是你先能看懂普通写法。

---

## 十七、对象简写语法

### 1. 简写属性 `shorthand property`

对应视频时间：`01:05:58`

如果变量名和属性名一样：

```js
const message = 'good job';

const object5 = {
  message: message
};
```

可以简写成：

```js
const object5 = {
  message
};
```

---

### 2. 简写方法 `shorthand method`

对应视频时间：`01:07:11`

原本这样写：

```js
const object5 = {
  method: function function1() {
    console.log('method');
  }
};
```

可以简写成：

```js
const object5 = {
  method() {
    console.log('method');
  }
};
```

这叫方法简写语法。

## 十八、本课应该顺手补充理解的内容

### 1. 对象是“现实世界事物”的程序表示

比如商品、用户、分数、订单，都天然适合用对象表示。  
这就是为什么后面的项目越来越依赖对象。

---

### 2. 对象是前端项目里最常见的数据结构之一

以后你会经常看到：

1. 用户对象
2. 商品对象
3. 配置对象
4. 响应对象

所以这一课是非常核心的基础。

---

### 3. `JSON` 不是对象

很多初学者会把两者混为一谈。

要记住：

1. 对象是 JavaScript 里的数据结构
2. `JSON` 是一种文本格式

`JSON.stringify()` 的结果是字符串，不是对象。

---

### 4. `localStorage` 是浏览器提供的能力

它不是 JavaScript 语言本身的核心语法，而是浏览器环境提供的 Web API。  
初学阶段先把它当成“浏览器自带的存储工具”理解即可。

---

## 十九、常见错误与易混点

### 1. 把对象和数组混淆

对象：

```js
{ name: 'shirt', price: 799 }
```

数组：

```js
['shirt', 799]
```

对象靠属性名访问，数组靠索引访问。

---

### 2. 忘记属性和值之间要用冒号

错误：

```js
const product = {
  name = 'shirt'
};
```

正确：

```js
const product = {
  name: 'shirt'
};
```

---

### 3. 用点记法访问带 `-` 的属性

错误：

```js
product.delivery-time
```

正确：

```js
product['delivery-time']
```

---

### 4. 以为 `localStorage` 可以直接保存对象

错误思路：

```js
localStorage.setItem('score', score);
```

更稳妥的做法：

```js
localStorage.setItem('score', JSON.stringify(score));
```

---

### 5. 忘记 `JSON.parse()`

如果你读取后直接当对象用：

```js
const score = localStorage.getItem('score');
console.log(score.wins);
```

通常会出问题，因为取回来的还是字符串或 `null`。

---

### 6. 误以为两个内容一样的对象 `===` 会是 `true`

```js
{ message: 'hello' } === { message: 'hello' }
```

结果是 `false`，因为引用不同。

---

### 7. 误以为 `const` 会冻结对象内部

不会。  
`const` 只限制变量重新赋值，不限制对象内部属性修改。

---

## 二十、本课最重要的结论

这一课最应该牢固掌握的是：

1. 对象用于组织一组相关数据
2. 对象通过属性访问值
3. 点记法是默认写法，方括号记法用于特殊属性名或动态属性名
4. 对象里还能放对象和函数
5. 函数放进对象后就叫方法
6. `JSON.stringify()` 把对象变成字符串
7. `JSON.parse()` 把字符串变回对象
8. `localStorage` 适合保存刷新后还要保留的数据
9. `localStorage` 没有值时常得到 `null`
10. 对象是引用类型，赋值和比较都要特别小心

---

## 二十一、建议复习顺序

如果你要回看这课，建议按这个顺序：

1. 先理解对象是什么
2. 再练属性访问、修改、添加、删除
3. 再理解石头剪刀布分数对象
4. 再理解方括号记法和嵌套对象
5. 再学方法和内建对象
6. 然后学 `JSON`
7. 再学 `localStorage`
8. 最后理解 `null`、引用、解构和简写语法

这样会比从头到尾硬背更稳。

## 二十二、练习建议

### 1. 基础练习

1. 创建一个 `product` 对象，包含 `name` 和 `price`
2. 用点记法输出两个属性
3. 修改 `price`
4. 添加 `deliveryTime`
5. 删除 `deliveryTime`

### 2. 方括号记法练习

1. 创建一个带 `delivery-time` 属性的对象
2. 用方括号记法读取它
3. 用变量名动态读取对象属性

### 3. 嵌套对象练习

1. 创建一个 `product` 对象
2. 在里面加 `rating` 对象
3. 读取 `rating.stars`
4. 读取 `rating.count`

### 4. 方法练习

1. 创建一个对象
2. 在里面放一个方法
3. 调用这个方法
4. 再改成简写方法语法

### 5. `localStorage` 练习

1. 保存一个字符串到 `localStorage`
2. 读取它
3. 删除它
4. 再尝试保存一个对象，配合 `JSON.stringify()` 和 `JSON.parse()`

### 6. 引用练习

1. 创建 `object1`
2. 令 `object2 = object1`
3. 修改 `object1` 内部属性
4. 观察 `object2` 是否变化
5. 再创建一个内容相同的新对象，比较 `===`
