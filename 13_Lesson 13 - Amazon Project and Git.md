## 十三、用数组 + 对象保存产品数据

### 1. 产品列表为什么用数组

对应视频时间：`00:24:16`

因为产品是“一组商品”。

```js
const products = [ ... ];
```

对应视频时间：`00:24:52`

因为一个商品有多个属性：

### 3. 典型结构

```js
const products = [

    image: 'images/products/...',
    name: 'Black and Gray Athletic Cotton Socks',
    rating: {
      stars: 4.5,
      count: 87
    },
    priceCents: 1090
  }

```

这就是数组和对象组合形成的数据结构。

## 十四、数据结构 Data Structure

### 1. 本课定义

对应视频时间：`00:32:40`

课程用“数据结构”来描述：

> 如何组织和表示数据。

在这节课里，产品列表的数据结构是：
- 外层数组
- 内层对象
- 对象里又有嵌套对象 `rating`

---

### 2. 为什么所有产品对象要保持相同结构

对应视频时间：`00:32:00`

如果每个产品对象结构不一致：
- 生成 HTML 的代码会很难写
- 有些产品有 `priceCents`，有些没有
- 有些有 `rating.stars`，有些没有

所以课程强调：

> 所有对象尽量保持同样的属性结构。

这样循环处理时更稳定。

## 十五、为什么价格要用分 `cents` 保存

### 1. 课程建议

对应视频时间：`00:27:58`

不要直接把价格保存成美元小数，比如：

```js
10.90
```

而是保存成分：

```js
1090
```

---

### 2. 为什么这样更好

因为浮点数在 JavaScript 里做金钱计算时可能出现精度问题。

虽然本课没有深入讲精度误差，但这是一个非常常见的工程实践：

> 金钱尽量用最小货币单位存储。

例如：
- 美元 -> 分
- 人民币 -> 分

---

### 3. 显示时再转回去

显示到页面时再：

```js
priceCents / 100
```

这样数据存储更安全，显示也更方便控制。

---

## 十六、用 `forEach` + 模板字符串生成产品 HTML

### 1. 为什么不手写所有产品 HTML

对应视频时间：`00:20:11`

如果产品很多，复制粘贴大量 HTML 会导致：
- 代码冗长
- 难维护
- 每加一个商品都很麻烦

所以课程改成：

> 保存数据，再用 JavaScript 根据数据生成 HTML。

---

### 2. 基本遍历结构

对应视频时间：`00:33:35`

```js
products.forEach((product) => {
  const html = `...`;
});
```

每次循环处理一个 `product` 对象。

---

### 3. 模板字符串插值

课程里把这些内容替换成动态值：

```js
${product.name}
${product.image}
${product.rating.stars}
${product.rating.count}
${(product.priceCents / 100).toFixed(2)}
```

这一步非常关键，因为它说明：

> HTML 字符串不一定是写死的，它可以由数据动态生成。

---

## 十七、用累加器模式拼接全部产品 HTML

### 1. 典型写法

对应视频时间：`00:42:40`

```js
let productsHTML = '';

products.forEach((product) => {
  const html = `...`;
  productsHTML += html;
});
```

这里 `productsHTML` 就是一个字符串累加器。

---

### 2. 为什么这是累加器模式

因为：
- 先准备一个结果变量
- 每次循环往里加一点结果
- 最后得到完整结果

这和你前面 lesson 11 学的累加器模式完全一致，只是：
- 那时常常是数字或数组
- 现在是字符串

---

## 十八、用 DOM 把生成的 HTML 放到页面上

### 1. 先找到容器

对应视频时间：`00:44:13`

课程找到的是产品总容器：
- `products-grid`

然后给它加一个 JS 类：

```html
class="products-grid js-products-grid"
```

---

### 2. 再通过 `innerHTML` 替换内容

```js
document.querySelector('.js-products-grid').innerHTML = productsHTML;
```

这一步完成后：
- 页面上的所有产品卡片就来自 JS 生成的 HTML
- 而不再是手写在 HTML 文件里的那一大堆卡片代码

---

### 3. 这一步的意义

这一刻页面就真正变成“数据驱动”了。

以后你只要改数据：
- 新增产品
- 替换产品数据

页面就会跟着变。

## 十九、`toFixed(2)` 格式化价格

### 1. 为什么需要它


```js

```

但页面上希望显示：

```js

```

---

### 2. 用法

```js
(product.priceCents / 100).toFixed(2)
```

作用：
- 强制保留 2 位小数

补充理解：
- `toFixed(2)` 返回的是字符串
- 这里用于显示非常合适

## 二十四、HTML 数据属性 `data-*`

### 1. 什么是数据属性

对应视频时间：`01:04:56`

数据属性是 HTML 属性的一种，专门用来把自定义信息附加到元素上。

例如：

```html

```

---

### 2. 命名规则

对应视频时间：`01:06:11`

规则：
- 必须以 `data-` 开头
- 后面名字通常用短横线连接

例如：
- `data-product-id`
- `data-product-name`

这种写法叫 kebab-case。

---

### 3. 为什么这在本课特别有用

因为点击某个按钮时，代码需要知道：

> 你点的是哪个商品的按钮？

按钮本身就是最自然的“信息载体”。

所以课程把商品信息挂到按钮上。

## 二十五、`dataset` 读取数据属性

### 1. 基本用法

对应视频时间：`01:07:13`

```js
button.dataset.productId
```

这会读取：

```html
data-product-id="..."
```

---

### 2. 命名转换规则

对应视频时间：`01:08:38`

HTML 里：

```html
data-product-id
```

JavaScript 里：

```js
dataset.productId
```

也就是从 kebab-case 变成 camelCase。

这点很容易考，也很容易忘。

## 二十六、购物车中新增商品 vs 增加已有商品数量

### 1. 一开始的简单做法

对应视频时间：`01:09:13`

最初直接：

```js
cart.push({
  productId,
  quantity: 1
});
```

这样会导致重复添加同一个商品时，购物车里出现两条相同商品记录。

---

### 2. 正确思路

对应视频时间：`01:10:57`

课程给出的正确流程是：

1. 先检查商品是否已经在购物车里
2. 如果在，就 `quantity += 1`
3. 如果不在，就新增一项

---

### 3. 典型实现

```js
let matchingItem;

cart.forEach((item) => {
  if (productId === item.productId) {
    matchingItem = item;
  }
});

if (matchingItem) {
  matchingItem.quantity += 1;
} else {
  cart.push({
    productId,
    quantity: 1
  });
}
```

这段代码把 lesson 11 和 lesson 12 的很多内容都串起来了：
- 数组遍历
- 条件判断
- 对象更新
- truthy / falsy

## 二十七、为什么商品识别要用 `id` 而不是 `name`

### 1. 课程里的理由

对应视频时间：`01:15:44`

商品名称不一定唯一。

两个不同品牌商品可能：
- 名字很像
- 甚至完全同名

所以如果用 `name` 做唯一标识，会有风险。

---

### 2. 为什么 `id` 更适合

`id` 的目标就是：

> 唯一标识一条数据。

所以购物车里应该保存：

```js
productId
```

而不是商品名字。

这是一条非常有工程意义的规则。

---

## 二十八、计算右上角购物车总数量

### 1. 基本思路

对应视频时间：`01:20:01`

购物车数量不是：
- `cart.length`

因为 `cart.length` 只表示购物车里有几种商品，不表示总件数。

例如：
- 篮球 1 个
- T 恤 2 个

总件数应该是 `3`，但 `cart.length` 只是 `2`。

---

### 2. 正确做法

```js
let cartQuantity = 0;

cart.forEach((item) => {
  cartQuantity += item.quantity;
});
```

这又是一个累加器模式。

---

### 3. 为什么这一步很重要

它说明页面上的很多显示值并不是“直接存着的”，而是：

> 根据数据计算出来的。

这是一种非常重要的前端思维。

---

## 二十九、用 DOM 更新购物车数量显示

### 1. 找到显示数量的元素

对应视频时间：`01:22:40`

课程先在页面上定位购物车数量元素，再给它增加 JS 类名。

---

### 2. 更新页面内容

```js
document.querySelector('.js-cart-quantity').innerHTML = cartQuantity;
```

这样每次点击 Add to Cart 后，页面右上角数字都会更新。

---

### 3. 为什么初始值要改成 0

对应视频时间：`01:25:15`

因为页面初始时购物车应该是空的。

如果 HTML 里默认写了别的数字，就会和真实数据不一致。

这再次说明：

> 页面显示应该和数据状态保持一致。

## 三十、本课最重要的方法论：数据驱动页面

### 1. 从手写 HTML 到数据驱动

这一课最关键的变化是：

原来：
- 手工在 HTML 里写产品卡片

现在：
- 在 JS 里保存产品数据
- 循环生成产品 HTML
- 再渲染到页面

这就是“数据驱动页面”的雏形。

---

### 2. 为什么这比复制粘贴 HTML 更好

课程也明确展示了这一点：

如果要新增一个商品，只需要：
- 向数据数组新增一个对象

而不是：
- 复制一整大段 HTML
- 再逐处修改图片、名字、价格、评分

这大幅提升可维护性。

## 三十一、本课隐含但非常重要的工程习惯

### 1. 代码拆分

课程把代码逐渐拆成多个文件：
- `amazon.html`
- `scripts/amazon.js`
- `data/products.js`
- `data/cart.js`

这说明项目变大后，单文件思维会失效。

---

### 2. 先让功能跑通，再逐步重构

课程不是一开始就给出最完美结构，而是：
- 先在一个文件里做最小版本
- 再抽成 `products.js`
- 再抽出 `cart.js`
- 再用 `id` 替代 `name`

这很接近真实开发过程。

---

### 3. Git 贯穿整个开发过程

课程不是讲完 Git 就结束，而是不断在项目推进过程中：
- 查看改动
- 写 commit message
- 提交版本

这说明 Git 不是独立知识点，而是开发流程的一部分。

## 三十二、常见错误

### 1. 生成 HTML 后忘记插入页面

你可能已经生成了：

```js
productsHTML
```

但如果不做：

```js
innerHTML = productsHTML
```

页面不会变化。

---

### 2. 价格直接用小数存储

这在金钱计算里不稳妥。

更推荐：
- 存分
- 显示时再除以 100

---

### 3. 购物车总数量写成 `cart.length`

这会把“商品种类数”误当成“总商品件数”。

---

### 4. 用商品名当唯一标识

这在真实电商场景里不可靠。

---

### 5. 数据属性名和 `dataset` 名字对不上

HTML：

```html
data-product-id
```

JS：

```js
dataset.productId
```

不是：

```js
dataset.product-id
```

---

### 6. 脚本加载顺序写错

如果先加载 `amazon.js`，后加载 `products.js`，代码可能会因为 `products` 还不存在而报错。

---

### 7. 改了很多代码但不提交 Git

这样会失去版本节点，后面回退和定位问题都很痛苦。

## 三十四、复习版总结

这一课最核心的内容可以压缩成下面几句话：

- Git 用来跟踪代码改动，帮助你查看、撤销和保存版本历史。
- 真实网站开发常见流程是：先保存数据，再根据数据生成 HTML，最后让页面具有交互性。
- 商品列表适合用数组表示，单个商品适合用对象表示。
- 页面中的产品卡片不一定要手写，可以通过循环和模板字符串动态生成。
- 金钱数据更适合以最小单位存储，例如用分而不是美元小数。
- Add to Cart 的核心是：获取当前商品标识，把它加入购物车数据。
- 购物车里相同商品不应重复成多条，而应该增加数量。
- 用 `id` 识别商品比用 `name` 更可靠。
- 右上角购物车数量应该根据购物车数据计算出来，再更新到 DOM。
