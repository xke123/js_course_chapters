## 二、本课整体理解

Lesson 5 是 JavaScript 入门中一个真正的分水岭。

在前几课里，你虽然已经能：
- 写数字
- 写字符串
- 做计算
- 让按钮点击后执行代码

但这些代码大多是“一次性的”。

这一课开始，JavaScript 终于具备了一个非常关键的能力：

> 把一个值保存起来，后面继续使用，甚至不断更新它。

这就是变量的意义。

如果没有变量，很多交互功能都做不出来。  
比如购物车数量：
- 你点一次按钮，数量变成 1
- 再点一次，变成 2
- 再点一次，变成 3

这类“状态会变化”的功能，本质上都离不开变量。

## 三、什么是变量 Variable

### 1. 变量的定义

对应视频时间：`00:02:33`

课程把变量解释成：

> 一个可以在里面保存值的容器。

这个比喻非常适合初学者。

你可以先把变量想成一个带名字的盒子：
- 盒子里可以放一个值
- 以后你可以通过名字再找到它

这个值可以是：
- 数字
- 字符串
- 计算结果
- 后面还会学到对象、数组等

---

### 2. 为什么变量重要

因为变量让代码从“一次性表达式”变成了“可以记住信息的程序”。

没有变量时：
- `2 + 2` 算完就结束

有变量时：
- 你可以先把 `2 + 2` 的结果存起来
- 之后继续拿它做别的计算

这就是程序开始有“状态”的第一步。

---

## 四、用 `let` 创建变量

### 1. 最基本语法

对应视频时间：`00:02:56`

课程先用的是：

```js
let variable1 = 3;
```

这行代码可以拆成三部分：
- `let`：创建变量
- `variable1`：变量名
- `= 3`：把值 `3` 存进去

---

### 2. 这行代码在做什么

它的意思是：

> 创建一个叫 `variable1` 的变量，并把数字 `3` 保存进去。

以后你只要写：

```js
variable1
```

JavaScript 就会把它当作里面保存的值来看。

---

## 五、读取变量的值

### 1. 用 `console.log()` 查看变量内容

对应视频时间：`00:03:55`

例如：

```js
console.log(variable1);
```

控制台会显示变量里保存的值。

如果 `variable1` 里是 `3`，那就会输出 `3`。

---

### 2. 变量名不是字符串

补充知识点。

初学者容易混淆这两种写法：

```js
console.log(variable1);
console.log('variable1');
```

它们完全不同：
- 第一个是读取变量的值
- 第二个只是输出一段文字 `"variable1"`

这点一定要分清。

---

## 六、变量里可以保存什么

### 1. 保存数字

对应视频时间：`00:02:56`

例如：

```js
let score = 10;
```

---

### 2. 保存计算结果

对应视频时间：`00:05:12`

课程演示了：

```js
let calculation = 2 + 2;
```

这不是把 `2 + 2` 这段文字存进去，而是：

1. 先计算 `2 + 2`
2. 得到结果 `4`
3. 把 `4` 存进变量

这说明变量里保存的是“计算后的值”。

---

### 3. 保存字符串

对应视频时间：`00:08:12`

例如：

```js
let message = 'hello';
```

说明变量不只存数字，也能存文本。

---

### 4. 一个关键理解

变量不关心它“看起来像什么功能”，它只是保存一个值。  
这个值可以是：
- 数字
- 字符串
- 后面会学到的更复杂数据

变量的本质是“给值起名字”。

---

## 七、在计算中使用变量

### 1. 变量可以参与后续运算

对应视频时间：`00:06:08`

例如：

```js
let calculation = 2 + 2;
console.log(calculation + 2);
```

如果 `calculation` 的值是 `4`，那这段代码就相当于：

```js
4 + 2
```

结果就是 `6`。

---

### 2. 变量会被它保存的值“替换”

课程反复在强调这个思路：

> 在需要值的地方使用变量，JavaScript 会把变量理解成里面保存的值。

这就是变量能重复复用的基础。

---

### 3. 变量和表达式可以组合

例如：

```js
let price = 20;
let total = price + 5;
```

这说明变量和普通数字、普通表达式都能混着用。

---

## 八、变量命名规则

### 1. 变量名不能用保留字

对应视频时间：`00:09:12`

例如你不能把变量命名成：

```js
let let = 3;
```

因为 `let` 本身已经有特殊含义。

这类词叫保留字 `reserved words`。

---

### 2. 变量名不能以数字开头

对应视频时间：`00:09:28`

例如：

```js
let 2name = 'abc';
```

这是不行的。

但数字可以出现在中间或后面，例如：

```js
let item2 = 'abc';
```

---

### 3. 变量名不能有空格，也不能随便用特殊字符

对应视频时间：`00:09:48`

课程提到：
- 不能有空格
- 大多数特殊字符不行

常见可用的是：
- 字母
- 数字
- `$`
- `_`

---

### 4. 实际上最推荐怎么命名

补充知识点。

入门阶段你最稳妥的做法就是：
- 主要用英文单词
- 多个单词时用 `camelCase`
- 避免奇怪缩写

这样可读性最好。

---

## 九、分号 `;`

### 1. 分号表示一条指令结束

对应视频时间：`00:10:07`

课程把分号比作英文里的句号。

例如：

```js
let count = 0;
console.log(count);
```

每条语句后面加分号，可以帮助 JavaScript 清楚地区分不同指令。

---

### 2. JavaScript 有自动补分号机制，但课程仍建议手动写

对应视频时间：`00:11:34`

JavaScript 有 `semicolon insertion`，有时会自动补分号。  
但课程依然建议显式写出来。

这是个很好的习惯，因为：
- 可读性更清楚
- 更不容易踩某些边界坑

入门阶段跟着这个习惯走是对的。

---

## 十、赋值和重新赋值

### 1. 第一次把值放进变量：赋值 `assign`

对应视频时间：`00:13:33`

例如：

```js
let variable1 = 3;
```

这是第一次给变量值，叫赋值。

---

### 2. 后面改成别的值：重新赋值 `reassign`

对应视频时间：`00:12:23`

例如：

```js
variable1 = 5;
```

这表示：
- 不创建新变量
- 只是把原来的值改掉

---

### 3. 重新赋值时不能再写 `let`

对应视频时间：`00:13:57`

这是高频错误。

错误思路：

```js
let variable1 = 3;
let variable1 = 5;
```

这样会出问题，因为你是在尝试“再次创建同名变量”。

正确做法是：

```js
let variable1 = 3;
variable1 = 5;
```

---

### 4. 变量值会随代码执行顺序变化

对应视频时间：`00:12:58`

课程特意提醒了一个很关键的点：

如果你在上面先输出变量，然后在下面才重新赋值，那么上面的输出看到的还是旧值。

这说明：

> 代码是按顺序执行的，变量的值也会随着执行位置不同而不同。

---

## 十一、变量自增与累加

### 1. 最基础写法

对应视频时间：`00:14:44`

课程演示了：

```js
variable1 = variable1 + 1;
```

它的意思是：

1. 取出 `variable1` 当前的值
2. 加 `1`
3. 再把新结果存回 `variable1`

这就是最基础的“加一”。

---

### 2. 为什么这很重要

因为很多交互都靠这种模式：
- 购物车数量加一
- 分数增加
- 步骤计数
- 点击次数累加

这类“更新状态”的能力，本质上都是重新赋值。

---

## 十二、购物车数量项目

### 1. 项目核心思路

对应视频时间：`00:16:02`

课程做了一个简化版购物车数量功能。

目标很明确：
- 初始数量是 `0`
- 点按钮后增加数量
- 可以加 `1`
- 可以加 `2`
- 可以加 `3`
- 可以重置为 `0`

---

### 2. 先用变量保存 `cartQuantity`

对应视频时间：`00:21:18`

课程使用了：

```js
let cartQuantity = 0;
```

这就是整个功能的核心状态。

也就是说：

> 页面并不是“凭空知道购物车数量”，而是通过变量保存当前数量。

---

### 3. 用模板字符串显示数量

对应视频时间：`00:23:20`

课程把变量放进模板字符串里：

```js
`Cart quantity: ${cartQuantity}`
```

这样控制台显示的就是变量当前值，而不是写死的文字。

这也是前几课知识点的连接：
- 变量
- 模板字符串
- `console.log`

三者第一次真正配合起来。

---

### 4. 点击按钮后更新变量

对应视频时间：`00:25:10`

例如“加入购物车”按钮的核心逻辑就是：

```js
cartQuantity = cartQuantity + 1;
```

然后再：

```js
console.log(`Cart quantity: ${cartQuantity}`);
```

这就是“状态更新 + 显示新状态”的基础流程。

---

### 5. `+2`、`+3`、重置按钮

对应视频时间：`00:27:00` 到 `00:30:15`

这几个按钮只是同一个模式的不同版本：

加二：

```js
cartQuantity = cartQuantity + 2;
```

加三：

```js
cartQuantity = cartQuantity + 3;
```

重置：

```js
cartQuantity = 0;
```

这说明一旦你掌握了变量和重新赋值，很多交互按钮本质上都很类似。

## 十三、重新赋值的快捷写法

### 1. `+=`

对应视频时间：`00:30:38`

课程讲了：

```js
cartQuantity += 2;
```

它等价于：

```js
cartQuantity = cartQuantity + 2;
```

意思是：
- 把当前值加上 `2`
- 再保存回自己

---

### 2. `++`

对应视频时间：`00:31:52`

课程还讲了：

```js
cartQuantity++;
```

它等价于：

```js
cartQuantity += 1;
```

也等价于：

```js
cartQuantity = cartQuantity + 1;
```

这是“加一”最常见的简写之一。

---

### 3. 还有类似的其他快捷方式

对应视频时间：`00:32:35`

课程提到还有：
- `-=`
- `*=`
- `/=`
- `--`

你现在至少要知道：

> 它们本质上都是“先计算，再重新赋值给自己”的简写。

---

## 十四、变量命名约定

### 1. `camelCase`

对应视频时间：`00:32:59`

课程强调，JavaScript 变量名通常使用驼峰命名法 `camelCase`。

例如：

```js
cartQuantity
```

规则是：
- 第一个单词小写
- 后面每个单词首字母大写

---

### 2. 其他常见命名风格

对应视频时间：`00:33:43`

课程顺带介绍了：

`PascalCase`

```txt
CartQuantity
```

`kebab-case`

```txt
cart-quantity
```

`snake_case`

```txt
cart_quantity
```

其中课程的结论很清楚：
- JavaScript 变量通常用 `camelCase`
- HTML / CSS / 文件名里经常会看到 `kebab-case`

---

### 3. 名字不要太短，也不要太长

对应视频时间：`00:35:05`

课程这个建议非常实用。

太短：

```js
let c = 0;
```

不容易看懂。

太长：

```js
let quantityOfProductsInCart = 0;
```

又会让代码太啰嗦。

所以好的变量名通常是：
- 能表达含义
- 又不过度冗长

`cartQuantity` 就是个很好的例子。

---

## 十五、`let`、`const`、`var`

### 1. `let`

对应视频时间：`00:35:42`

`let` 用来创建“后续可以重新赋值”的变量。

例如：

```js
let count = 0;
count = 1;
```

这种会变化的值很适合用 `let`。

---

### 2. `const`

对应视频时间：`00:36:16`

`const` 也能创建变量，但它的值不能重新赋值。

例如：

```js
const value = 3;
```

之后如果你再写：

```js
value = 5;
```

就会报错。

---

### 3. 为什么课程建议默认优先用 `const`

对应视频时间：`00:37:35`

课程给出的理由很重要：

> `const` 让代码更安全、更容易理解。

因为看到 `const`，你就知道：
- 这个值不会被改
- 后面读代码时更稳定

所以最佳实践是：

1. 默认先用 `const`
2. 只有确定这个变量后面需要改，才改用 `let`

---

### 4. `var`

对应视频时间：`00:38:28`

`var` 是 JavaScript 里更老的变量写法。

课程提到：
- 旧代码里会看到
- 现代新代码里通常不推荐继续使用

你现在只要知道它存在，以及现代入门阶段优先使用：
- `const`
- `let`

就够了。

---

## 十六、`typeof` 和变量

### 1. 可以检查变量里保存的值类型

对应视频时间：`00:39:10`

例如：

```js
console.log(typeof variable2);
```

如果 `variable2` 里是数字，就会输出：

```txt
number
```

如果某个变量里保存字符串，就会输出：

```txt
string
```

---

### 2. `typeof` 检查的是“变量里的值”，不是变量名本身

这点和前面讲的一样很重要。

变量本身只是一个名字，真正有类型的是它里面保存的值。

---

## 十七、本课最容易出错的地方

### 1. 重新赋值时又写了 `let`

这是高频错误。

---

### 2. 把变量名写错大小写

JavaScript 区分大小写，`cartQuantity` 和 `cartquantity` 不是同一个名字。

---

### 3. 变量名起得太随便

太短会难懂，太长会难读。

---

### 4. 明明不会变的值还用 `let`

这不是语法错误，但会降低代码可读性。

---

### 5. 误以为 `const` 代表“完全不能存在”

不是。  
`const` 代表变量创建后不能重新赋值，不是说不能使用。

---

### 6. 分不清“赋值”和“比较”

虽然这节课还没正式讲比较运算，但现在先提醒：

```js
=
```

不是同一个东西。

---

## 十八、本课最重要的知识点总结

1. 变量是保存值的容器
2. `let name = value;` 可以创建变量
3. 变量里可以保存数字、字符串和计算结果
4. 变量可以在后续代码里重复使用
5. 变量的值可以重新赋值
6. `variable = variable + 1` 是基础自增模式
7. `+=` 和 `++` 是常见快捷写法
8. JavaScript 变量命名通常用 `camelCase`
9. 默认优先用 `const`
10. 只有需要修改时再使用 `let`
11. `var` 是旧写法，现代代码少用
12. `typeof` 可以检查变量里保存值的类型

## 二十一、给初学者的最终提醒

这节课最重要的收获，不是会写几行 `let`，而是开始理解：

> 程序不只是“执行一次表达式”，而是能够记住状态，并随着用户操作不断更新状态。

购物车数量只是第一个例子。  
后面你学到的输入框内容、待办列表、页面数据、对象属性，本质上都会回到这个核心能力：

> 把值保存起来，再在合适的时候读取和更新它。
